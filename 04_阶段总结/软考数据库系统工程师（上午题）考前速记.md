## 二、数据库系统核心速记

### 1. 三级模式两级映射（必考）

| 层次 | 作用 |
|---|---|
| 外模式 | 用户视图 |
| 概念模式 | 全局逻辑结构 |
| 内模式 | 物理存储 |

| 两级映射 | 保证 |
|---|---|
| 外模式/概念模式 | 逻辑独立性 |
| 概念模式/内模式 | 物理独立性 |

> [!tip] 高频考法
> “修改视图实现的是逻辑独立性”。2017 上午真题考过。

### 2. ER 模型（上午+下午核心）

#### 联系类型

| 类型 | 含义 |
|---|---|
| 1:1 | 一对一 |
| 1:N | 一对多 |
| M:N | 多对多 |

#### ER 转关系模式

| ER 元素 | 转换 |
|---|---|
| 实体 | 关系模式 |
| 属性 | 字段 |
| 联系 | 外键/新关系 |

多对多必须单独建表。

例如：

```text
学生 —— 选课 —— 课程
```

转为：

```text
学生(学号,...)
课程(课程号,...)
选课(学号,课程号,...)
```

> [!warning] 易错点
> 多对多不能直接加外键，必须新建联系表。

|---|---|---|

### 3. 笛卡尔积属性问题（高频）

2017 真题考过。

若：

```text
R(A,B,C)
S(A,C,E)
```

则：

```text
R × S
```

重复属性必须保留表名前缀：

```text
(R.A,B,R.C,S.A,S.C,E)
```

> [!warning] 易错点
> 同名属性不会自动合并。

## 四、函数依赖与规范化（上午最核心）

### 1. 函数依赖判断

#### 传递依赖

```text
A→B
B→C
⇒ A→C
```

#### 部分依赖

```text
(A,B)→C
A→C
```

则 C 对组合键部分依赖。

### 2. 范式速记（必背）

| 范式   | 消除          |
| ---- | ----------- |
| 1NF  | 字段不可再分      |
| 2NF  | 消除部分依赖      |
| 3NF  | 消除传递依赖      |
| BCNF | 每个决定因素都是候选键 |
| 4NF  | 消多值依赖       |

### 3. 判断套路（考试专用）

1. 找候选键。
2. 看非主属性是否：
   - 部分依赖主键 → 不满足 2NF
   - 传递依赖主键 → 不满足 3NF
3. 若决定因素不是候选键 → 不满足 BCNF。

### 4. BCNF 高频陷阱

2019 上午真题：

- 最小依赖集：右边必须单属性。
- 左边不一定单属性。

> [!warning] 高频错误
> “最小依赖集左边必须单属性”是错误说法。

### 5. 规范化口诀（极重要）

> [!tip] 口诀
> - 1NF 看原子
> - 2NF 消部分
> - 3NF 消传递
> - BCNF 看决定

## 五、SQL 高频考点

### 1. DDL / DML / DCL

| 类型 | 命令 |
|---|---|
| DDL | CREATE、ALTER、DROP |
| DML | SELECT、INSERT、UPDATE、DELETE |
| DCL | GRANT、REVOKE |

### 2. GROUP BY 高频套路

```sql
SELECT dept, COUNT(*)
FROM student
GROUP BY dept;
```

原则：`SELECT` 中只能出现聚合函数和 `GROUP BY` 字段，之外不能出现普通字段。

### 3. NULL 高频陷阱

正确写法：

```sql
WHERE score IS NULL
```

不能写：

```sql
= NULL
```

### 4. EXISTS 与 IN

| 写法 | 特点 |
|---|---|
| IN | 小结果集 |
| EXISTS | 大结果集 |

### 5. DELETE / TRUNCATE / DROP

| 命令 | 删除数据 | 删除结构 |
|---|---|---|
| DELETE | √ | × |
| TRUNCATE | √ | × |
| DROP | √ | √ |

### 6. 主键 vs 外键

| 类型 | 是否重复 | 是否为空 |
|---|---|---|
| 主键 | 不允许 | 不允许 |
| 外键 | 允许 | 允许 |

## 六、事务与并发控制（超高频）

### 1. ACID（必背）

| 属性 | 含义 |
|---|---|
| A | 原子性 |
| C | 一致性 |
| I | 隔离性 |
| D | 持久性 |

### 2. 三大并发问题

| 问题    | 含义    |
| ----- | ----- |
| 脏读    | 读未提交  |
| 不可重复读 | 两次读不同 |
| 幻读    | 行数变化  |
| 丢失修改  | 修改被覆盖 |

### 3. 隔离级别（必考）

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---|---|---|---|
| Read Uncommitted | 有 | 有 | 有 |
| Read Committed | 无 | 有 | 有 |
| Repeatable Read | 无 | 无 | 有 |
| Serializable | 无 | 无 | 无 |

2018 上午真题考过。

### 4. 锁机制

| 锁 | 含义 |
|---|---|
| S 锁 | 共享锁（读锁） |
| X 锁 | 排它锁（写锁） |

| 情况 | 是否兼容 |
|---|---|
| S-S | √ |
| S-X | × |
| X-X | × |

### 5. 两段锁协议（必背）

- 加锁阶段：只能加锁。
- 解锁阶段：只能解锁。

### 6. 可串行化调度

判断方法：

1. 找冲突操作：不同事务、同一数据、至少一个写。
2. 画优先图：
   - 有环 → 不可串行化
   - 无环 → 可串行化

2017 上午真题考过。

## 十二、上午题高频易错点

| 易错点 | 正确理解 |
|---|---|
| NULL 用 = 判断 | 用 IS NULL |
| BCNF = 3NF | BCNF 更严格 |
| 共享锁能写 | 共享锁只能读 |
| 检查点防死锁 | 检查点只提升恢复效率 |
| Read Committed 防幻读 | 不能 |

## 十四、考前最后 30 分钟必背

1. **ACID**：原子、一致、隔离、持久。
2. **范式口诀**：1NF 原子；2NF 消部分；3NF 消传递；BCNF 看决定。
3. **锁兼容**：S-S 兼容；S-X 冲突；X-X 冲突。
4. **恢复**：已提交 REDO；未提交 UNDO。
5. **两级映射**：外概 → 逻辑独立；概内 → 物理独立。
