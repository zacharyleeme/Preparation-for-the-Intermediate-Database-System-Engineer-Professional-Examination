# SQL 题答题模板

> 适用范围：软考中级《数据库系统工程师》下午题中的 **SQL 建表约束 / 查询补全 / 视图 / 触发器 / 存储过程 / 游标 / 事务控制** 题型。
>
> 题源口径：基于 **2016～2023 可靠真题池** 的真实题型整理。
> - 2016～2019：依据 `05_真题库/文本化补证_2016-2019/` 中已落盘 Markdown 文本直接核对。
> - 2020～2023：依据 `05_真题库` 中 PDF 可抽取文本层的真实题干片段核对。
> - 2024～2025：当前仍为扫描型 PDF，**不纳入本模板的题型统计依据**。
>
> 使用目标：把 SQL 题压缩成“**先看输出要求，再定主表连接；先看约束，再填语句骨架；先保业务正确，再补并发与触发逻辑**”的固定作答链。

### 2. 结论先记住

2016～2023 的 SQL 题并不是随便拼语法，而是稳定围绕下面几条主线反复出：

1. **CREATE TABLE：主键 / 外键 / 唯一 / 检查 / 级联**
2. **SELECT 多表连接：查明细、查统计、查最大、查未发生**
3. **CREATE VIEW：聚合统计、补零、二次复用**
4. **INSERT / UPDATE / DELETE：批量插入、联动修改、删除迁移**
5. **TRIGGER：自动维护数量、金额、缺货、违规记录**
6. **PROCEDURE / TRANSACTION：业务流程补全、隔离级别、提交回滚**
7. **CURSOR / EXISTS / NOT IN / NOT EXISTS / OUTER JOIN：按业务语义选结构**

一句话：

> **SQL 题比的不是“背了多少关键字”，而是你能不能先识别业务语义，再把它翻译成对的语句骨架。**

## 二、SQL 题考场固定顺序

### 通用 6 步主链

1. **先读题目要什么结果**：输出哪些列、哪些条件、怎么排序  
2. **再确定主表与连接链**：数据分别来自哪些表  
3. **再判断题型**：建表 / 查询 / 视图 / 触发器 / 过程  
4. **再补核心结构**：`JOIN` / `WHERE` / `GROUP BY` / `HAVING` / `ORDER BY` / `REFERENCES`  
5. **最后检查业务边界**：空值、未发生、全部、最大值、级联、回滚  
6. **收尾核对列名与方向**：主外键方向、统计口径、排序方向、是否少表别名

### 通用标准句序

> 先根据题意明确输出列、筛选条件和排序要求；  
> 再确定涉及的基础表及其连接关系；  
> 若题目是统计类，再补上分组属性和聚合函数；  
> 若题目是“未参加 / 未售出 / 全部都满足”类，再判断使用 `NOT IN`、`NOT EXISTS` 或外连接；  
> 若题目是建表或触发器，则优先落实主键、外键、检查约束和触发时机；  
> 最后检查 SQL 是否与业务语义一一对应。

## 三、先背下来的总判断口诀

### 1. 看到“定义实体完整性、参照完整性、其他完整性约束”
> 先想：`PRIMARY KEY`、`REFERENCES`、`UNIQUE`、`CHECK`、`ON DELETE CASCADE`

### 2. 看到“查询详细情况”
> 先想：多表连接 + 明细列，不要急着 `GROUP BY`

### 3. 看到“统计每个……”
> 先想：`GROUP BY` + `COUNT/SUM/AVG`

### 4. 看到“最多 / 最大 / 最高”
> 先想：聚合后 `HAVING = (子查询 max/聚合结果)`

### 5. 看到“未参加 / 未售出 / 没安排过”
> 先想：反查，优先 `NOT IN` / `NOT EXISTS`

### 6. 看到“即使没有也要输出”
> 先想：`LEFT OUTER JOIN` 或 `UNION` 补零

### 7. 看到“给所有……插入一条记录”
> 先想：`INSERT INTO ... SELECT ...`

### 8. 看到“自动维护 / 自动增加 / 自动记录”
> 先想：`TRIGGER`

### 9. 看到“失败回滚 / 正确提交 / 隔离级别”
> 先想：事务过程题，先看 `BEGIN TRANSACTION / COMMIT / ROLLBACK`

## 四、题型 1：CREATE TABLE 建表约束题模板

### 真题稳定考法
2016、2017、2018、2019、2020、2021、2022、2023 都核到了这类题。

### 固定写法模板

```sql
CREATE TABLE 表名(
  主码列 数据类型 PRIMARY KEY,
  普通列 数据类型,
  外码列 数据类型 REFERENCES 被参照表(被参照列),
  约束列 数据类型 CHECK (取值条件),
  唯一列 数据类型 UNIQUE
);
```

### 若是组合主键

```sql
CREATE TABLE 表名(
  A 数据类型,
  B 数据类型,
  ...,
  PRIMARY KEY (A, B),
  FOREIGN KEY (A) REFERENCES 表1(A),
  FOREIGN KEY (B) REFERENCES 表2(B)
);
```

### 直接可套写的判断模板

> 若某属性唯一标识一条记录，则应设为主键。  
> 若某属性取值必须来自另一表的主键，则应设为外键引用。  
> 若题干给出枚举取值、上下界或时间范围，则应补 `CHECK` 约束。  
> 若题干明确“删除主表时从表也要删除”，则应补 `ON DELETE CASCADE`。

### 高频失分点
- 主键和外键方向写反；
- 忘记 `CHECK`；
- 把“唯一但可空/不可空”的语义和主键混掉；
- 看见“删除时级联”却没写 `CASCADE`。

## 五、题型 2：多表连接明细查询模板

### 真题稳定考法
2020 订单详细信息、2019 学生学院查询、2016/2017/2018/2022/2023 都有同类基础。

### 识别信号
- “输出订单号、客户姓名、商品名称、单价、数量、金额”
- “输出学号、姓名、学院名”
- “输出编号、名称、总量”

### 通用模板

```sql
SELECT 表1.列1, 表2.列2, 表3.列3
FROM 表1, 表2, 表3
WHERE 表1.键 = 表2.键
  AND 表2.键 = 表3.键
  AND 其他筛选条件
ORDER BY 排序列 ASC|DESC;
```

### 直接可套写思路

> 先找输出列分别来自哪张表；  
> 再找这些表之间用什么主外键连接；  
> 若题目只是查明细而非统计，不要乱加 `GROUP BY`；  
> 最后按题目要求加排序。

### 高频失分点
- 少连一张表；
- 把筛选条件写进 `HAVING`；
- 排序方向写反；
- 明细查询误用聚合。

## 六、题型 3：聚合统计 / 视图题模板

### 真题稳定考法
2016、2017、2019、2020、2022 高频；2023 也有“全部课程”这类集合语义。

### 常见题干信号
- “统计每个……数量”
- “订单个数 / 平均每单数量”
- “每天销售数量”
- “在项目中已使用零件的代码和使用量”

### 通用模板

```sql
CREATE VIEW 视图名 AS
SELECT 分组列, 聚合函数(统计列) AS 别名
FROM 基表
WHERE 条件
GROUP BY 分组列;
```

### 多表统计模板

```sql
SELECT A.编号, B.名称, COUNT(*) AS Num, AVG(A.数量) AS AvgNum
FROM A, B
WHERE A.外键 = B.主键
GROUP BY A.编号, B.名称;
```

### “先建总表，再复用”的写法
这在 2017、2020 很典型：

```sql
CREATE VIEW 总表 AS
SELECT 键, COUNT(*) AS amount
FROM 明细表
GROUP BY 键;

CREATE VIEW 明细统计表 AS
SELECT ...
FROM 维表, 总表
WHERE 维表.键 = 总表.键;
```

### 高频失分点
- `SELECT` 里有非聚合列却没进 `GROUP BY`；
- 统计口径错，把“每单”写成“每商品”；
- 忘记视图只是结果，不要再加无关列。

## 七、题型 4：查“未发生 / 未参加 / 未售出”模板

### 真题稳定考法
2016、2017、2020、2021、2022、2023 都核到了这类反查语义。

### 最稳模板 1：`NOT IN`

```sql
SELECT 主表.键, 主表.名称
FROM 主表
WHERE 主表.键 NOT IN (
  SELECT DISTINCT 从表.键
  FROM 从表
  WHERE 条件
);
```

### 最稳模板 2：`NOT EXISTS`

```sql
SELECT A.键, A.名称
FROM A
WHERE NOT EXISTS (
  SELECT *
  FROM B
  WHERE B.外键 = A.键
    AND 条件
);
```

### 如何选
- 子查询结果明确是一列键值集合：优先 `NOT IN`
- 需要和外层逐行关联：优先 `NOT EXISTS`
- 题目强调“全部都满足 / 不存在任何一个不满足”：通常双重 `NOT EXISTS`

### 高频失分点
- 忘记 `DISTINCT` 导致语义混乱；
- 外层键和内层键没对应上；
- “未售出”写成了“售出数量为 0”，但实际上原表里根本没记录。

## 八、题型 5：查“最多 / 最大 / 第一名”模板

### 真题稳定考法
2016 查询调度次数最多的汽车、2017 查询当天销售最多的商品。

### 通用模板

```sql
SELECT 分组对象, COUNT(*) AS Num
FROM 表
GROUP BY 分组对象
HAVING COUNT(*) >= ALL (
  SELECT COUNT(*)
  FROM 表
  GROUP BY 分组对象
);
```

或者题目已经给出固定骨架时，常见填法是：

```sql
HAVING COUNT(*) = (
  SELECT MAX(cnt)
  FROM (
    SELECT COUNT(*) AS cnt
    FROM 表
    GROUP BY 分组对象
  ) T
)
```

### 极简思路

> 先按对象分组求次数；  
> 再让当前组的次数与“所有组中的最大次数”比较。

### 高频失分点
- 忘记 `GROUP BY`；
- 子查询统计对象和外层不一致；
- “最多”只写 `ORDER BY DESC`，但没处理并列最多。

## 九、题型 6：补零 / 即使没有也要输出 模板

### 真题稳定考法
2019 俱乐部人数视图、2023 间接先修课为空也要输出、2022 已用/未用零件合并。

### 模板 1：`LEFT OUTER JOIN`

```sql
SELECT A.键, B.列
FROM A LEFT OUTER JOIN B
ON A.键 = B.键;
```

适合：
- 主表必须全部保留；
- 从表可能不存在匹配记录；
- 不存在时显示空值。

### 模板 2：`UNION` 补零

```sql
SELECT 键, 名称, COUNT(*) AS Num
FROM 关联表, 主表
WHERE 连接条件
GROUP BY 键, 名称
UNION
SELECT 键, 名称, 0
FROM 主表
WHERE 键 NOT IN (SELECT DISTINCT 键 FROM 关联表);
```

适合：
- 题目明确要求“没有记录时人数为 0 / 数量为 0”；
- 已给出一段统计结果，再补无记录对象。

### 高频失分点
- 该保留主表却写成内连接；
- `UNION` 两边列数不一致；
- 补零分支忘记同样的列顺序。

## 十、题型 7：`INSERT INTO ... SELECT` 批量插入模板

### 真题稳定考法
2023 “给所有学生插入新开课选课记录”。

### 通用模板

```sql
INSERT INTO 目标表(列1, 列2)
SELECT 源表.列1, 常量或另一列
FROM 源表
WHERE 条件;
```

### 直接可套写

> 若题目要求“给所有……新增一条记录”，一般不是一条一条插，而是 `INSERT INTO ... SELECT`。  
> 目标表写目标列，`SELECT` 负责批量产出每一行要插入的值。

### 高频失分点
- 把 `VALUES` 和 `SELECT` 混写；
- 忘记列顺序对应；
- 常量列和来源列位置错位。

## 十一、题型 8：UPDATE / DELETE 联动题模板

### 真题稳定考法
2022 仓库迁移：先改再删；2021 竞赛取消：依赖级联删除；2016/2018/2020 过程里也有联动更新。

### 通用模板

```sql
UPDATE 表
SET 列 = 新值
WHERE 条件;

DELETE FROM 表
WHERE 条件;
```

### 解题原则

> 先看业务是否要求“迁移后删除旧记录”；  
> 若有外键依赖，先更新从表引用，再删主表；  
> 若题干说“根据已定义参照完整性可直接删除”，说明它在暗示级联约束已经建立。

### 高频失分点
- 先删主表导致外键冲突；
- 只更新主表，漏掉关联明细；
- 不分“人工先删明细”与“依赖级联自动删”。

## 十二、题型 9：触发器模板

### 真题稳定考法
2016 奖金维护、2017 缺货单、2018 不良记录、2019 学院人数维护、2021 自动补货、2022 余额变更通知、2023 工资表业务规则。

### 通用结构

```sql
CREATE TRIGGER 触发器名
AFTER|BEFORE INSERT OR UPDATE OR DELETE ON 表名
REFERENCING NEW ROW AS nrow OLD ROW AS orow
FOR EACH ROW
BEGIN
  -- 条件判断
  -- 更新 / 插入 / 删除
END;
```

### 三类高频业务模板

#### 1. 维护汇总值

```sql
UPDATE 汇总表
SET 数量 = 数量 + 1
WHERE 主键 = nrow.外键;
```

#### 2. 满足条件时自动插记录

```sql
INSERT INTO 记录表
SELECT ...
FROM 源表
WHERE 条件;
```

#### 3. 违规时阻断或记违规

```sql
IF 条件 THEN
  INSERT INTO 异常表 ...;
END IF;
```

### 先判触发时机
- 新增记录后维护统计：常见 `AFTER INSERT`
- 修改某列后联动处理：常见 `AFTER UPDATE`
- 删除记录后回收数量：常见 `AFTER DELETE`
- 要阻止非法操作：常见 `BEFORE`

### 高频失分点
- 应该用 `NEW` 却写成 `OLD`；
- 触发时机错，把“写入后统计”写成 `BEFORE`；
- 触发器里忘了按当前行关联条件过滤。

## 十三、题型 10：存储过程 / 事务控制模板

### 真题稳定考法
2016 出入库、2017 抢红包、2018 结算、2019 余额汇总、2020 游标更新、2021 销售类别过程、2022 日终汇总、2023 工资计算。

### 通用结构

```sql
CREATE PROCEDURE 过程名(参数列表)
BEGIN
  SET TRANSACTION ISOLATION LEVEL 隔离级别;
  BEGIN TRANSACTION;

  -- 查询 / 插入 / 更新 / 删除

  IF error THEN
    ROLLBACK;
    return -1;
  END IF;

  COMMIT;
  return 0;
END;
```

### 先判隔离级别的思路
- 题目说“只要正确性，不考虑并发性能”：偏高隔离级别
- 题目说“保证信息不丢失，同时最大并发度”：偏较低隔离级别但仍要满足业务
- 题目出现“读取后又更新”：要特别警惕脏读、不可重复读、丢失更新

### 直接可套写的过程题思路

> 先看该过程的目标是“读汇总”“写记录”“更新库存”还是“批量维护”；  
> 再判断哪些语句一旦失败必须整体撤销；  
> 因此要在关键 SQL 后检查执行状态，出错就 `ROLLBACK`，成功才 `COMMIT`。

### 高频失分点
- 忘记回滚；
- 隔离级别与题目“并发度 / 正确性”要求相反；
- 只插记录不更新主业务表；
- 把“业务成功返回”写在 `COMMIT` 前面。

## 十四、题型 11：游标题模板

### 真题稳定考法
2017、2020 都明确核到了基于游标的过程补全。

### 通用结构

```sql
DECLARE 游标名 CURSOR FOR
SELECT ...
FROM ...
WHERE ...;

OPEN 游标名;
FETCH 游标名 INTO 变量;
WHILE NOT error
BEGIN
  -- 对当前行处理
  FETCH 游标名 INTO 变量;
END
CLOSE 游标名;
```

### 什么时候会考游标
- 需要逐行处理临时表数据；
- 每读一行就更新一次目标表；
- 不适合一句聚合 SQL 直接做完。

### 高频失分点
- `OPEN / FETCH / CLOSE` 顺序错；
- 没有在循环末尾继续 `FETCH`；
- 把逐行变量和列名混淆。

## 十五、题型 12：双重 `NOT EXISTS` / “全部都修完”模板

### 真题稳定考法
2023 “查询选课表中已选修了全部课程的学生”。

### 这是经典关系除法题

### 通用模板

```sql
SELECT A.键, A.名称
FROM A
WHERE NOT EXISTS (
  SELECT *
  FROM 全集B
  WHERE NOT EXISTS (
    SELECT *
    FROM 关系表R
    WHERE R.A键 = A.键
      AND R.B键 = B.键
  )
);
```

### 记忆句

> **不存在一门课，是这个学生没有选过的。**

这句话翻成 SQL 就是双重 `NOT EXISTS`。

### 高频失分点
- 少写一层 `NOT EXISTS`；
- 外层和内层关联键没对上；
- 把“全部”误写成“存在”。

---

## 十六、考场最小检查清单

写完 SQL 题后，至少强制检查这 10 项：

- [ ] 输出列是不是都能从 `FROM` 中的表拿到？
- [ ] 多表之间主外键连接是不是写全了？
- [ ] 统计题的非聚合列是不是都进了 `GROUP BY`？
- [ ] “未发生”类题是不是用了反查结构？
- [ ] “即使没有也要输出”是不是保留了主表全量？
- [ ] 建表题有没有同时检查 `PRIMARY KEY / REFERENCES / CHECK / UNIQUE`？
- [ ] 触发器里 `NEW/OLD` 用对了吗？
- [ ] 事务题里出错是不是会 `ROLLBACK`？
- [ ] 排序列和升降序是不是符合题意？
- [ ] 列名、表名、别名有没有前后不一致？

---

## 十七、最后压缩成一句话

> **SQL 题固定顺序：先看输出，再定表；先连关系，再补条件；统计就分组，未发生就反查，必须保留主表就外连或补零，自动维护就触发器，整体一致就事务回滚。**
