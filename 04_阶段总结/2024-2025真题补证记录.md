## 四、正式结论

> [!warning] 当前补证结论
> 2024、2025 真题 PDF 已确认存在，但均为扫描图像型 PDF；当前环境缺少可用 OCR 引擎，OpenClaw 图像模型也因 API key / 模型能力问题不可用，因此本轮不能可靠产出 2024、2025 真题全文文本。

- 2024 真题文件存在，共 27 页，全部为图像页。
- 2025 真题文件存在，共 34 页，全部为图像页。
- 普通 PDF 文本抽取不可用。
- 已完成图片渲染，为后续 OCR 留好了中间页图。

当前不能写成事实的内容：

## 五、后续建议

### 方案 A：安装本地 OCR 后继续

推荐安装或启用以下任一 OCR 路线：

1. `tesseract` + 中文语言包 `chi_sim`
2. `ocrmypdf`
3. `paddleocr`
4. `rapidocr_onnxruntime`

安装完成后，可直接对已渲染图片目录继续 OCR。

### 方案 B：修复 OpenClaw image model

当前图像模型失败原因为 API key 配置无效：`q`。若修复 `agents.defaults.imageModel` 或相关 image provider 配置，可继续使用已渲染图片页进行批量 OCR。

### 方案 C：人工摘录关键题型

如果暂不安装 OCR，也可人工打开 2024、2025 PDF，先摘录：

- 下午题 1～5 题题型
- 是否包含数据库设计题
- 是否包含范式 / 分解题
- 是否包含 SQL 题
- 是否包含恢复 / 并发 / 性能优化题

然后再回填到整体备考规划与真题训练计划。
