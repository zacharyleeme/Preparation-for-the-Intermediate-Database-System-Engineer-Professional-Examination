## 第 1 题：数据库设计

### 错题 1-1：问题 1｜实体联系图设计

#### 错题原题

某新能源汽车公司为了提升效率，需开发一个汽车零件采购系统。需求包括：

- 记录供应商信息，包括供应商的名称、地址和一个电话。
- 记录零件信息，包括零件的编码、名称和价格。
- 记录车型信息，包括车型的编号、名称和规格。
- 记录零件采购信息，某个车型的某种零件可以从多家供应商采购，某种零件也可以被多个车型采用，某家供应商也可以供应多种零件，还包括采购数量和采购日期。

问题：根据描述补充实体联系图，不增加新的实体。

#### 错误答案

未作答 / 未给出实体联系图或等价文字说明。

#### 正确答案

应在供应商、零件、车型之间建立“采购”联系，体现三方参与关系，并将采购数量、采购日期作为采购联系的属性。

#### 答案解析

题干没有要求新增实体，而是要求补充实体联系图。这里的采购不是单独实体，而是供应商、零件、车型之间的联系：某车型采购某零件，可以来自某供应商。因此该联系至少需要同时关联：

- 供应商
- 零件
- 车型

并在联系上记录：

- 采购数量
- 采购日期

本题容易漏答图形题。下午题如果不能画图，也应写出“供应商—采购—零件—车型”的文字等价结构，至少保住部分采分点。

### 错题 1-2：问题 2｜采购关系空缺、主键和外键

#### 错题原题

根据概念结构设计阶段完成的实体联系图，得出如下关系模式：

```text
供应商(名称，地址，电话)
零件(编码，名称，价格)
车型(编号，名称，规格)
采购(车型编号，供应商名称，( a )，( b )，采购日期)
```

问题：补充 `(a)`、`(b)` 两处空缺，并标注主键和外键完整性约束。

#### 错误答案

```text
(a)：零件编号
(b)：采购数量
该关系无主键，车型编号为外键引用车型(编号)，供应商名称为外键引用供应商(名称)，零件编号为外键引用零件(编号)
```

#### 正确答案

```text
(a)：零件的编码 / 零件编码
(b)：采购数量
主键：组合主键(车型编号，供应商名称，零件编码，采购日期)
外键：车型编号，供应商名称，零件编码
```

#### 答案解析

`采购`关系记录的是“某车型、某供应商、某零件、某日期”的采购事实。由于同一车型、供应商、零件可能在不同日期发生多次采购，所以采购日期也要进入组合主键。

本题失分点：

1. 将标准字段“零件编码”写成“零件编号”，字段名不严谨。
2. 更严重的是写成“无主键”。关系模式必须有候选键/主键，否则无法唯一标识采购记录。
3. 外键方向基本有意识，但应引用零件(编码)，不是零件(编号)。

避免方法：看到多对多/三元联系转换为关系模式时，优先检查“参与实体主键 + 联系区分属性”是否构成组合主键。

### 错题 1-3：问题 3｜新增门店销售关系设计

#### 错题原题

新增需求：记录车型在全国门店的销售情况，门店信息包括门店编号、地址和电话，销售包括销售数量和销售日期等。

问题：

1. 在图中体现门店信息及其车型销售情况，并标明新增的实体和联系及其重要属性。
2. 给出新增加的关系模式并标注主键和外键完整性约束。

#### 错误答案

```text
门店信息（门店编号，地址，电话）
销售（销售单号，门店编号，零件标号，销售数量，销售日期）
门店信息的主键为门店编号，无外键；销售关系模式的主键为销售单号，外键为门店编号，引用门店信息(门店编号)
```

#### 正确答案

```text
门店(门店编号，地址，电话)
主键：门店编号

销售(门店编号，车型编号，销售数量，销售日期)
组合主键：(门店编号，车型编号，销售日期)
外键：门店编号、车型编号
```

#### 答案解析

题干说的是“车型在全国门店的销售情况”，销售对象是车型，不是零件。因此销售关系中应包含 `车型编号`，而不是 `零件标号`。

本题关键采分点：

- 新增实体：门店。
- 新增联系：门店销售车型。
- 销售关系的外键应同时指向门店和车型。
- 销售记录可用 `(门店编号，车型编号，销售日期)`作为组合主键。

避免方法：新增需求先圈出“谁和谁发生联系”。本题是“门店—车型”，不是“门店—零件”。

## 第 2 题：规范化与 BCNF

### 错题 2-1：问题 1｜冗余属性与异常

#### 错题原题

维修配件使用表：

```text
维修配件使用表(车牌号，维修时间，配件编码，配件名称，配件供应商，配件仓库编码，仓库地址，维修配件数量)
```

已知：

- 同一辆车在同一次维修情况下可能需要多种维修配件。
- 一种配件只能存放于一个配件仓库。
- 一种配件只能由一个配件供应商提供。
- 维修时间精确到秒。

问题：该表存在数据冗余，请给出具体冗余属性，并说明因此会出现哪些异常。

#### 错误答案

```text
属性车牌号和配件编码为冗余属性，数据冗余会导致表在更新删除时出现异常，不利于数据的长期统计和统一管理。
```

#### 正确答案

```text
配件编码 → 配件名称，配件供应商，配件仓库编码
配件仓库编码 → 仓库地址

冗余属性：配件名称、配件供应商、配件仓库编码、仓库地址。
异常：插入异常、更新异常、删除异常。
```

#### 答案解析

判断冗余属性时，不是看哪些字段重复出现，而是看哪些字段被其他非主属性函数决定，导致在多条维修记录中反复保存。

- `配件编码`决定配件的名称、供应商、仓库编码。
- `配件仓库编码`决定仓库地址。

因此当同一配件出现在多条维修记录中，配件名称、供应商、仓库编码、仓库地址会重复存储，形成冗余。

异常说明：

- 插入异常：没有维修记录时，可能无法单独录入某配件或仓库信息。
- 更新异常：供应商或仓库地址变化时，需要更新多条记录，容易不一致。
- 删除异常：删除最后一条维修记录时，可能连带丢失配件或仓库信息。

避免方法：先写函数依赖，再由依赖找冗余；不要凭直觉把主键候选字段写成冗余属性。

### 错题 2-2：问题 2｜BCNF 分解

#### 错题原题

维修配件使用表是否满足 BCNF？如果不满足，请对其进行模式分解，使分解后的关系模式满足 BCNF，并标记出主键、外键。

#### 错误答案

```text
不满足 BCNF。
可以将当前关系模式拆分为：
配件（配件编码，配件名称，配件供应商）
配件存储（配件仓库编码，仓库地址，配件名称）
配件使用（车牌号，配件名称，维修配件数量，维修时间）

配件关系模式中主键为配件编码，无外键；
配件存储关系模式中配件仓库编码为主键，配件名称为外键引用配件（配件名称）；
配件使用关系模式中主键为车牌号，外键为配件名称，引用配件（配件名称）。
```

#### 正确答案

```text
不满足 BCNF。

1. 维修配件使用表(车牌号，维修时间，配件编码，维修配件数量)
   主键：车牌号 + 维修时间 + 配件编码
   外键：配件编码

2. 配件信息表(配件编码，配件名称，配件供应商，配件仓库编码)
   主键：配件编码
   外键：配件仓库编码

3. 仓库信息表(配件仓库编码，仓库地址)
   主键：配件仓库编码
```

#### 答案解析

本题分解应围绕函数依赖的决定因素展开：

- `配件编码` 是配件信息的决定因素，因此配件信息表应以配件编码为主键，并保留配件名称、供应商、仓库编码。
- `配件仓库编码` 是仓库地址的决定因素，因此仓库信息表应以配件仓库编码为主键。
- 维修使用事实需要记录车牌号、维修时间、配件编码、数量，主键应为三者组合：车牌号 + 维修时间 + 配件编码。

错误答案的问题：

1. 用“配件名称”作为外键，不稳定；名称可能重复或变更。
2. 配件使用表缺少配件编码，导致不能准确关联配件。
3. 主键只写车牌号，无法区分同一车不同时间、不同配件的维修记录。

避免方法：BCNF 分解中，优先保留“编码类字段”作为主键/外键，不用名称字段承担标识作用。

## 第 3 题：SQL 填空

### 错题 3-1：问题 1｜COURSE 表完整性约束

#### 错题原题

补全创建课程表 COURSE 的 SQL：

```sql
CREATE TABLE COURSE(
Cno CHAR(4) PRIMARY KEY，
Cname CHAR(30) ( a )，
Cpno CHAR(4) references ( b )， ( c )
Ccredit INT
);
```

要求定义实体完整性约束、参照完整性约束，以及其他完整性约束。已知课程名取值唯一。

#### 错误答案

```text
(a)：not null
(b)：on SC(Cno)
(c)：not null
```

#### 正确答案

```text
(a)：unique
(b)：COURSE
(c)：Cno
```

等价理解：`Cpno CHAR(4) REFERENCES COURSE(Cno)`。

#### 答案解析

题干明确“课程名取值唯一”，因此 `Cname` 的约束应是 `UNIQUE`，不是单纯 `NOT NULL`。

`Cpno` 表示先修课课程号，先修课本身也是课程，因此它应引用同一个课程表 `COURSE` 的主键 `Cno`，属于自引用外键。

避免方法：

- “取值唯一” → `UNIQUE`
- “先修课课程号” → 引用 `COURSE(Cno)`
- 不要把先修课引用到选课表 `SC`。

### 错题 3-2：问题 2｜为所有学生插入新课程选课记录

#### 错题原题

有一门课程号为 `C036` 的新开课，要求所有学生选修。该课的基本信息已经录入课程表 COURSE 中，现需在选课表 SC 中插入该课的选课记录。

```sql
( d ) INTO SC(Sno, ( e ))
VALUES SELECT Sno，( f ) FROM ( g )
```

#### 错误答案

```text
(d)：insert
(e)：Cno
(f)：Cno
(g)：COURSE
```

#### 正确答案

```text
(d)：INSERT
(e)：Cno
(f)：'C036'
(g)：STUDENT
```

#### 答案解析

目标是“所有学生都选修课程 C036”。因此要从 `STUDENT` 表取出每个学生的 `Sno`，并为每个学生插入固定课程号 `'C036'`。

正确语义是：

```sql
INSERT INTO SC(Sno, Cno)
SELECT Sno, 'C036' FROM STUDENT;
```

错误答案从 `COURSE` 表取 `Cno`，会变成“从课程表取课程号”，不能为所有学生生成选课记录。

避免方法：`INSERT SELECT` 题先判断“行数来自哪个表”。本题行数来自所有学生，所以 `FROM STUDENT`。

### 错题 3-3：问题 4｜查询选修全部课程的学生

#### 错题原题

查询选课表中已选修了全部课程的学生，要求输出学号和姓名。

```sql
SELECT Sno，Sname FROM STUDENT WHERE NOT EXISTS
(SELECT * FROM ( l )
WHERE ( m )
(SELECT* FROM ( n )
WHERE ( o ) );
```

#### 错误答案

```text
(l)：SC
(m)：SC.Cno not in
(n)：COURSE
(o)：SC.Cno = COURSE(Cno)
```

#### 正确答案

```text
(l)：COURSE
(m)：NOT EXISTS
(n)：SC
(o)：SC.Sno=STUDENT.Sno AND SC.Cno=COURSE.Cno
```

#### 答案解析

“选修了全部课程”是典型关系除法问题，常用双重 `NOT EXISTS`：

> 不存在这样一门课程：该学生没有选它。

逻辑结构是：

```sql
SELECT Sno, Sname
FROM STUDENT
WHERE NOT EXISTS (
  SELECT *
  FROM COURSE
  WHERE NOT EXISTS (
    SELECT *
    FROM SC
    WHERE SC.Sno = STUDENT.Sno
      AND SC.Cno = COURSE.Cno
  )
);
```

外层枚举所有课程 `COURSE`，内层检查当前学生是否选了该课程。错误答案把 `SC` 和 `COURSE` 位置写反，也没有建立学生、课程、选课三者的完整关联。

避免方法：关系除法口诀：

> 全部 X：不存在一个 X，使得不存在对应记录。

## 第 4 题：存储过程、触发器与并发控制

### 错题 4-1：问题 1｜工资计算存储过程

#### 错题原题

补全月底 25 日计算某员工最终工资的存储过程：

```sql
CREATE PROCEDURE SalaryCalculation( ( a ) empld char(8), IN iYear number(4), IN iMonth number(2))
DECLARE
attendance number(14, 2);
merit number(14, 2);
overtime number(14, 2);
base number(14, 2);
all salary number(14, 2);
BEGIN
SELECT attendance_wage, merit_pay, overtime_wage INTO ( b )
FROM SALARY WHERE Eid=empld FOR UPDATE;
SELECT Jbase_salary INTO :base FROM EMPLOYEES T1, ( c )
WHERE T1.Jid=T2.Jid AND T1.Eid= empld;
all_salary := attendance + merit + overtime + base;
UPDATE SALARY SET salary = :all_salary
WHERE ( d ) AND year = iYear AND month = iMonth;
( e )
EXCEPTION WHEN OTHERS THEN ( f ) END
```

#### 错误答案

```text
(a)：空
(b)：:overtime
(c)：JOB_LEVELS T2
(d)：Eid=empid
(e)：空
(f)：rollback
```

#### 正确答案

```text
(a)：IN
(b)：:attendance, :merit, :overtime
(c)：JOB_LEVELS T2
(d)：eid=empId
(e)：COMMIT
(f)：ROLLBACK
```

#### 答案解析

该过程输入参数包括员工编号、年份、月份，因此 `(a)` 应为 `IN`。

`SELECT attendance_wage, merit_pay, overtime_wage INTO ...` 一次查询了 3 个字段，所以 `INTO` 后也必须对应 3 个变量：

```sql
:attendance, :merit, :overtime
```

工资更新成功后需要提交事务，因此 `(e)` 是 `COMMIT`；异常时回滚，因此 `(f)` 是 `ROLLBACK`。

避免方法：SQL/过程题要检查“SELECT 几列 → INTO 几个变量”，数量必须一致。

### 错题 4-2：问题 2｜工资表修改触发器

#### 错题原题

员工工资表修改规则：只能在每月 25 日的上班时间进行。补全触发器：

```sql
CREATE TRIGGER CheckBusinessRule ( g )
INSERT OR DELETE OR ( h ) on SALARY FOR EACH ( i ) BEGIN
IF (TO_CHAR(sysdate, 'DD')＜>( j ) OR (to_number(TO_CHAR(sysdate,'HH24')) ( k ) BETWEEN 8 AND 18) THEN
Raise_Error;
END IF:
END:
```

#### 错误答案

```text
(g)：空
(h)：update
(i)：row
(j)：25
(k)：not
```

#### 正确答案

```text
(g)：BEFORE
(h)：UPDATE
(i)：row
(j)：'25'
(k)：NOT
```

#### 答案解析

该触发器用于阻止非法修改，应该在插入、删除、更新真正执行前检查，所以触发时机是 `BEFORE`。

其余空：

- `UPDATE`：覆盖更新操作。
- `FOR EACH row`：行级触发器。
- 日期为 25 日。
- 时间不在 8～18 点之间时抛异常，因此是 `NOT BETWEEN 8 AND 18`。

避免方法：凡是“防止非法写入/修改”的业务规则触发器，通常优先考虑 `BEFORE`。

### 错题 4-3：问题 3｜并发问题与 2PL

#### 错题原题

当某月计算一位员工最终工资时，同一时间人事部门对该员工执行了奖励 2000 元的事务操作。

问题：

1. 请说明该事务调度存在哪种并发问题？
2. 采用 2PL 是否可以解决该并发问题？是否会产生死锁？

#### 错误答案

```text
(1) 该事务调度存在计算最终工资事务读锁与人事部门奖罚事务写锁冲突异常。
(2) 采用2PL可以将事务按顺序进行读写，从而避免读操作与写操作参数冲突异常。2PL不能消除死锁，但是可以让事务锁按顺序执行，减少锁等待时间。
```

#### 正确答案

```text
(1) 存在丢失更新：人事部门奖惩事务更新后的最终工资，可能被计算最终工资事务后写入的结果覆盖。
(2) 2PL 可以解决该并发问题，但可能会产生死锁。
```

#### 答案解析

本题不是泛泛问“锁冲突”，而是问具体并发异常。两个事务都可能写同一员工工资：

- 人事事务把工资加 2000。
- 工资计算事务随后根据旧数据计算并写回最终工资。

如果工资计算事务的写入覆盖了人事事务的奖励结果，就发生“丢失更新”。

2PL（两段锁协议）通过加锁和持锁到合适阶段，可以保证冲突操作串行化，从而避免丢失更新。但多个事务互相等待对方释放锁时，仍可能产生死锁。

避免方法：看到“一个事务更新结果被另一个事务覆盖”，直接识别为“丢失更新”。

## 第 5 题：故障恢复与日志

### 错题 5-1：问题 3｜Undo 补偿日志格式

#### 错题原题

根据业务逻辑填写日志记录空白 `(a)`；给出 Undo 恢复的补偿日志记录，填写 `(b)` 和 `(c)`。

已知：某次组装过程出库 12 个配件，安装 6 个。T3 负责将未使用的配件重新入库。

#### 错误答案

```text
(a)：14
(b)：＜T3, I, 14, 8>
(c)：＜T3，abort>
```

#### 正确答案

```text
(a)：14
(b)：＜T3,I,8＞
(c)：＜T3,abort＞
```

#### 答案解析

`(a)` 的数值判断正确：出库 12 个，安装 6 个，剩余 6 个重新入库，对应日志中的结果值为 14。

问题出在 `(b)` 的补偿日志格式。题干已说明：

- `＜Ti, D, V1, V2＞` 表示事务 Ti 将数据项 D 从 V1 改为 V2。
- `＜Ti，D, V＞` 表示将事务 Ti 回滚数据项 D 的值回滚到 V。

Undo 补偿日志应使用回滚格式，所以应写：

```text
＜T3,I,8＞
```

而不是普通更新日志格式 `＜T3, I, 14, 8＞`。

避免方法：恢复日志题先区分两种格式：

- 正常更新：`<T, D, old, new>`
- Undo 回滚补偿：`<T, D, restored_value>`
