# 软考数据库系统工程师（上午题）分块打印版

> 用途：打印 / 分模块背诵 / 临考前按弱项分块回看
>
> 范围：仅保留当前模拟考中反复失分、易混、送分却做错的内容
>
> 标记：`⭐高频失分` `⚠必须拿分`

---

## 模块一：数据库基础

### A. 必背定义卡

| 知识点 | 一句话秒记 | 常见错法 | 快速纠偏 |
|---|---|---|---|
| ⭐ 关系 / 元组 / 属性 | 表=关系，行=元组，列=属性 | 把行当关系、把列当记录 | 看到“行/列”先翻译成“元组/属性” |
| ⚠ 关系型数据库 | 关系型数据库是**表的集合**，表是**记录的集合** | 把“表/记录”顺序背反 | 直接整句背，不拆开记 |
| ⭐ 主键 / 候选键 / 外键 | 主键从候选键中选；参照完整性靠外键 | 把参照完整性记成主键实现 | 看到“参照完整性”直接想“外键” |
| ⚠ NULL | NULL 是空值，不是 0，不是空串 | 见 NULL 就往数值空字符串上靠 | 题干有 NULL，先想“三值逻辑” |
| ⭐ 实体完整性 / 参照完整性 | 实体完整性靠主键；参照完整性靠外键 | 主外键职责混淆 | 先看“约束的是本表还是跨表” |

### B. 高频陷阱

- `行`、`列`、`表` 三词最容易误判，先映射后作答。
- `候选键没找对`，后面范式、依赖、分解全会错。
- `NULL` 相关题不要用日常直觉，要回到数据库语义。

### C. 秒记结论

- 行=元组；列=属性；表=关系
- 关系型数据库是表的集合，表是记录的集合
- 参照完整性靠外键
- 实体完整性靠主键
- NULL 是空值，不是 0，不是空串

### D. 送分题清单 ⚠

- 关系 / 元组 / 属性
- 主键 / 外键
- NULL
- 实体完整性 / 参照完整性

## 模块二：SQL

### A. 必背语法卡

| 知识点 | 一句话秒记 | 常见错法 | 快速纠偏 |
|---|---|---|---|
| ⭐ DELETE / DROP | DELETE 删行；DROP 删对象 | 把删记录写成 DROP | 先判断删的是“数据”还是“表/对象” |
| ⭐ GRANT / REVOKE | GRANT 授权；REVOKE 回收 | 语法方向写反 | 记“给权 GRANT，收权 REVOKE” |
| ⭐ SQL 顺序 | `SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY` | WHERE / GROUP BY 顺序反 | 先筛行，再分组，再筛组 |
| ⭐ 子查询 | 先看返回单值还是多值 | 看见子查询就默认多行 | 出现聚合函数常先判断为单值 |
| ⭐ 联结 | 逗号+where 本质是连接筛选 | 把连接结果理解错 | 先看连接条件，再看结果列 |
| ⚠ NULL 判断 | NULL 不能直接用 `=` 比较 | 把 `= NULL` 当真条件 | 看到 NULL 先想 `IS NULL` |

### B. 高频陷阱

- `删除数据` 和 `删除对象` 绝不是一回事。
- `授权` 和 `回收权限` 最容易因为词眼熟而写反。
- `GROUP BY` 前面一定先有 `WHERE`。
- 子查询题不要先选，先判“返回 1 个值还是 1 组值”。

### C. 秒记结论

- DELETE 删行，DROP 删对象
- GRANT 授权，REVOKE 回收
- `SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY`
- 子查询先判单值还是多值
- 联结先看连接条件
- NULL 判断优先想 `IS NULL`

### D. 考场快检

- 题目问的是删“行”还是删“表”？
- 是回收权限还是继续授权？
- 是先筛选再分组，还是你背反了？
- 子查询有没有 `SUM / COUNT / AVG / MAX / MIN`？

## 模块三：事务并发

### A. 必背概念卡

| 知识点 | 一句话秒记 | 常见错法 | 快速纠偏 |
|---|---|---|---|
| ⭐ ACID | C=持久性/持续性，不是一致性 | 把 C 记成 consistency | 记“提交后不丢”=持久性 |
| ⭐ 脏读 | 读到了未提交数据 | 和不可重复读混 | 有“未提交”三个字基本就是脏读 |
| ⭐ 不可重复读 | 同一事务两次读同一数据，值变了 | 和幻读混 | 关注“同一行值变化” |
| ⭐ 幻读 | 两次查询记录数变了 | 和不可重复读混 | 关注“行数变化/多出少了记录” |
| ⭐ 共享锁 / 排它锁 | S 锁共读；X 锁独占 | 以为 S 锁也能写 | 看有没有写操作 |
| ⭐ 一级/二级/三级封锁协议 | 一级防丢失修改；二级防脏读；三级防不可重复读 | 协议和异常对不上 | 背成固定链条 |
| ⭐ 可串行化 | 并发正确性的核心标准 | 只会背名词，不知道作用 | 看到“一致性”先想它 |
| ⭐ 系统故障恢复 | 常用 UNDO + REDO | 只选 undo 或 redo | 先看故障类型，再看提交状态 |
| ⭐ WAL | 日志先写，后写数据 | 顺序背反 | 见日志恢复题先想 WAL |

### B. 高频陷阱

- “撤销后另一个事务读到了旧值” → 很可能是脏读。
- “同一事务前后两次读值变了” ≠ 幻读。
- “共享锁”不是万能锁，只能保证共享读。
- 恢复题先分 `事务故障 / 系统故障 / 介质故障`，再谈 undo/redo。

### C. 秒记结论

- ACID 中 C 别背错：这里考的是持久性/持续性
- 脏读=读未提交
- 不可重复读=同一行值变了
- 幻读=记录数变了
- S 锁共读，X 锁独占
- 一级锁防丢失修改
- 二级锁防脏读
- 三级锁防不可重复读
- 可串行化是并发正确性核心
- 系统故障恢复常用 UNDO+REDO
- WAL：日志先写

### D. 考场快检

- 题干说的是“值变了”还是“记录数变了”？
- 有没有“未提交”“撤销”“回滚”字样？
- 问的是锁类型，还是封锁协议，还是隔离异常？
- 恢复题先看故障类型，不要直接选操作。

## 模块四：组成原理

### A. 必背定义卡

| 知识点 | 一句话秒记 | 常见错法 | 快速纠偏 |
|---|---|---|---|
| ⭐ PC / IR | PC 存下一条指令地址；IR 存当前指令 | 下一条地址错选 IR | 一看“下一条地址”就选 PC |
| ⭐ ALU / 加法器 | 加法器属于 ALU 相关部件 | 把部件和寄存器混掉 | 看到“算术逻辑单元”先想运算器 |
| ⭐ Cache | 解决 CPU 与主存速度不匹配 | 错选主存与外存不匹配 | 看到 Cache 就先想 CPU↔主存 |
| ⭐ DMA | 主存与外设直接交换，CPU 参与少 | 和中断、查询混 | 比较 CPU 参与度：查询最多，DMA 最少 |
| ⭐ RISC / CISC | RISC：寄存器多、硬布线；CISC：微程序控制常见 | 把两者特征背反 | 看“微程序控制”多半偏 CISC |
| ⚠ 磁盘碎片整理 | 提高文件访问效率，不修坏扇区 | 以为能修坏扇区/整理内存 | 看到“坏扇区”直接排除 |

### B. 高频陷阱

- `下一条地址` 和 `当前指令` 是最典型送分坑。
- `Cache` 题不要被“外存”带跑，核心是 CPU 与主存之间。
- `DMA / 中断 / 程序查询` 本质比的是 CPU 忙不忙。
- `RISC / CISC` 喜欢反着出题。

### C. 秒记结论

- PC 看下条，IR 看这条
- Cache 解决 CPU 与主存速度差
- DMA 最省 CPU
- 程序查询最累，中断次之，DMA 最省
- RISC 常硬布线、寄存器多
- CISC 常微程序控制
- 磁盘碎片整理不修坏扇区

### D. 考场快检

- 题干问的是“当前指令”还是“下一条地址”？
- 是 CPU↔主存，还是主存↔外存？
- 是考“DMA 是什么”，还是考“哪个最少占 CPU”？
- 有“微程序控制”字样时先想 CISC。

## 模块五：数据结构

### A. 必背定义卡

| 知识点 | 一句话秒记 | 常见错法 | 快速纠偏 |
|---|---|---|---|
| ⭐ 栈 / 队列 | 栈=LIFO；队列=FIFO | 先进后出、先进先出背反 | 栈后队先 |
| ⭐ 二叉树遍历 | 先序根前，中序根中，后序根后 | 根位置记混 | 只盯根的位置 |
| ⭐ 折半查找 | 必须有序，且通常适合静态表 | 只记“快”，忘记前提 | 看到“有序、少插删”就选折半 |
| ⭐ Huffman 树 | 最优二叉树，可用于编码/解码 | 把编码解码当错误项 | 一看到“最优二叉树”就联想 Huffman |
| ⚠ 哈希函数 | 变长输入映射到固定长度输出 | 把固定/变长方向说反 | 看到“固定长输入变长输出”直接排除 |

### B. 高频陷阱

- 栈队列最容易因为熟悉而掉以轻心。
- 二叉树遍历题核心不是整棵树，而是“根在前中后哪儿”。
- 折半查找不是“任何查找都更快”，前提是有序且不常改。
- Huffman 经常用“错误的是”反向出题。

### C. 秒记结论

- 栈后进先出，队列先进先出
- 先序根前，中序根中，后序根后
- 折半查找必须有序
- Huffman 是最优二叉树，可编码解码
- 哈希函数：变长输入 → 固定长输出

### D. 考场快检

- 这题问的是“访问顺序”还是“存取规则”？
- 有没有“有序、顺序存储、少插删”的前提？
- 题目是不是在用“错误的是”反向卡 Huffman？
