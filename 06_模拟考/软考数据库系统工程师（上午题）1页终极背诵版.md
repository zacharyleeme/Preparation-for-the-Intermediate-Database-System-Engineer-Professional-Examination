# 软考数据库系统工程师（上午题）1页终极背诵版

> 目标：最后 30 分钟高密度速翻
>
> 原则：只留最容易错、最该拿分、最能直接提分的结论

---

## ① 数据库基础

- 表=关系；行=元组；列=属性。
- 关系型数据库是**表的集合**，表是**记录的集合**。
- 实体完整性靠**主键**，参照完整性靠**外键**。
- NULL 是**空值**，不是 0，不是空串。
- 候选键先找，再做范式判断。

## ② SQL

- DELETE 删**行**，DROP 删**对象**。
- GRANT 授权，REVOKE 回收。
- `SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY`
- NULL 判断优先想 `IS NULL`。
- 连接题先看**连接条件**，再看结果列。
- 子查询先判：返回**单值**还是**多值**。

## ③ 事务并发

- ACID 中 C 别背错：这里考的是**持久性/持续性**。
- 脏读=读未提交。
- 不可重复读=同一行前后值变了。
- 幻读=前后查询记录数变了。
- S 锁共读，X 锁独占。
- 一级锁防丢失修改；二级锁防脏读；三级锁防不可重复读。
- 可串行化是并发正确性的核心。
- 系统故障恢复常用 `UNDO + REDO`。
- WAL：**日志先写，数据后写**。

## ④ 范式 / 依赖（必须插进最后一页）

- 有“非主属性对码的传递依赖” ⇒ 最高 **2NF**。
- `所有属性完全依赖主键` ⇒ 至少 **2NF**。
- BCNF：**每个函数依赖左部都必须是码**。
- 范式题固定顺序：**先找候选键，再看部分依赖，再看传递依赖**。
- 无损连接题别死背一个条件，警惕“错误的是”。

## ⑤ 组成原理

- PC 存**下一条指令地址**；IR 存**当前指令**。
- Cache 解决 **CPU 与主存** 速度不匹配。
- DMA：主存与外设直接交换，**最省 CPU**。
- 程序查询最忙，中断次之，DMA 最省。
- RISC 常硬布线、寄存器多；CISC 常微程序控制。
- 磁盘碎片整理是提访问效率，不修坏扇区。

## ⑥ 数据结构

- 栈=LIFO；队列=FIFO。
- 先序根前，中序根中，后序根后。
- 折半查找必须：**有序 + 少插删**。
- Huffman 是最优二叉树，可编码解码。
- 哈希函数：**变长输入 → 固定长输出**。

## ⑦ 网络 / 送分杂项

- ADSL：**双绞线 + 电话网**。
- Telnet 默认端口：**23**。
- OLTP=联机事务处理；OLAP=联机分析处理。
- CAP：一致性、可用性、分区容错性，**最多满足两个**。
- 应用层数据单元通常是**消息**。

## ⑧ 最后扫题提醒

- 看到“错误的是 / 不正确的是 / 不包含”先反向。
- 看到 SQL 先找：动词、WHERE、GROUP BY、子查询返回值。
- 看到事务题先判：脏读 / 不可重复读 / 幻读。
- 看到范式题先做：**找键**。
- 看到组成原理题先抓关键词：PC / IR / Cache / DMA。
- 看到熟题也别秒点，先确认题干有没有“否定词”。

---

## ⑨ 一口气背诵串联版

- 表关系、行元组、列属性；主键实体、外键参照、NULL 空值。
- DELETE 删行，DROP 删表；GRANT 给权，REVOKE 收权。
- SQL 顺序：SELECT FROM WHERE GROUP BY HAVING ORDER BY。
- 脏读未提交，不重是值变，幻读是行变。
- 一锁防丢失，二锁防脏读，三锁防不重读。
- WAL 先日志，系统故障 UNDO+REDO。
- 候选键先找；传递依赖到 2NF；左部全是码才 BCNF。
- PC 看下条，IR 看当前；Cache 调 CPU 主存，DMA 最省 CPU。
- 栈后队先，根前中后，折半有序，Huffman 最优。
- ADSL 双绞线电话网，Telnet 23，OLTP 事务，OLAP 分析，CAP 最多两全。
