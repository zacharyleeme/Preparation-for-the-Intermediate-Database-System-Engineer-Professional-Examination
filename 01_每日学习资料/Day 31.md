### 2.3 今日核心知识框架

#### 主线 A：计算机组成与体系结构

| 高频点 | 必须会的判断 |
|---|---|
| Cache | 命中率越高，平均访问时间越接近 Cache 访问时间 |
| 流水线 | 理想吞吐率提高，但单条指令执行时间不一定缩短 |
| 存储层次 | 寄存器最快，Cache 次之，主存再后，外存最慢 |
| DMA | 适合高速外设与内存直接交换数据，减少 CPU 干预 |
| 总线 | 数据总线传数据，地址总线决定可寻址范围，控制总线传控制信号 |
| 校验码 | 奇偶校验能发现部分错误，海明码可定位并纠正单比特错误 |

#### 主线 B：数据结构与算法

| 高频点 | 必须会的判断 |
|---|---|
| 栈 | 后进先出，常用于递归、表达式求值、括号匹配 |
| 队列 | 先进先出，常用于缓冲、调度、层次遍历 |
| 二叉排序树 | 中序遍历得到递增序列 |
| 哈希查找 | 平均查找效率高，但冲突处理会影响性能 |
| 排序 | 快排平均快但最坏可退化；堆排序最坏仍为 `O(nlogn)` |
| 图 | 最小生成树关注连通无向图；拓扑排序用于有向无环图 |

#### 主线 C：操作系统

| 高频点 | 必须会的判断 |
|---|---|
| 进程与线程 | 进程是资源分配基本单位，线程是调度基本单位 |
| 死锁 | 互斥、占有并等待、不可剥夺、循环等待四条件 |
| 调度算法 | 时间片轮转适合分时系统，短作业优先可能导致长作业饥饿 |
| 分页存储 | 页号 + 页内地址，页表完成逻辑地址到物理地址转换 |
| 段式存储 | 按程序逻辑模块划分，段长可变 |
| PV 操作 | P 表示申请 / 等待，V 表示释放 / 唤醒 |

### 2.4 必备结论

- Cache 平均访问时间通常按“命中时间 + 未命中代价”思路估算。
- 地址总线位数决定最大可寻址空间。
- 流水线提高系统吞吐率，但不等于每条指令单独变快。
- DMA 的核心价值是减少 CPU 在大量 I/O 传输中的直接参与。
- 栈是后进先出，队列是先进先出。
- 二叉排序树中序遍历结果有序。
- 快速排序平均时间复杂度为 `O(nlogn)`，最坏可为 `O(n²)`。
- 死锁产生的四个必要条件必须同时具备。
- 分页存储存在内部碎片，分段存储更贴近逻辑结构。
- 事务 2PL 题中，后续事务必须等待前一事务释放冲突锁后再进入。

### 2.5 高频考点拆解

#### 2.5.1 Cache 平均访问时间

**核心概念：** CPU 访问数据时，若 Cache 命中则直接从 Cache 取，未命中才访问主存。  
**考法：** 给出命中率、Cache 访问时间、主存访问时间，计算平均访问时间。  
**高频误区：** 把主存访问时间和 Cache 访问时间简单相加，忽略命中率。  
**快速判断提示：** 命中率越高，平均访问时间越接近 Cache 时间。

#### 2.5.2 流水线吞吐率

**核心概念：** 流水线将指令执行拆成多个阶段，多条指令可重叠执行。  
**考法：** 判断流水线主要提升什么，或估算连续执行多条指令的总时间。  
**高频误区：** 认为流水线一定缩短单条指令完整执行时间。  
**快速判断提示：** 流水线主要提高吞吐率。

#### 2.5.3 树与遍历

**核心概念：** 树结构常考二叉树遍历、二叉排序树、哈夫曼树。  
**考法：** 根据遍历序列判断访问顺序，或判断二叉排序树性质。  
**高频误区：** 混淆前序、中序、后序的根节点访问位置。  
**快速判断提示：** 前序根在前，中序根在中，后序根在后。

#### 2.5.4 死锁四条件

**核心概念：** 死锁产生需要互斥、占有并等待、不可剥夺、循环等待同时成立。  
**考法：** 判断哪种策略破坏了哪个条件。  
**高频误区：** 只看到“等待”就判断死锁。  
**快速判断提示：** 没有循环等待，不一定是死锁。

#### 2.5.5 SQL 与 2PL 保温点

**核心概念：** Day 30 暴露的问题集中在“会模板但漏细节”。  
**考法：** 分组统计、反查、抢红包并发控制。  
**高频误区：** 分组列写多、判空字段不稳、后续事务等待没写。  
**快速判断提示：** **问谁按谁分组；反查判连接键空；后者等待再进入。**

### 2.7 标准答题模板 / 采分点提示

#### 模板1：SQL 分组统计题

```sql
SELECT 分组对象, COUNT(*) / AVG(...) / SUM(...)
FROM 表
WHERE 分组前条件
GROUP BY 分组对象
HAVING 聚合条件;
```

采分点：
1. 先明确“题目问谁”；
2. `GROUP BY` 只写必要分组列；
3. 聚合结果筛选写在 `HAVING`。

#### 模板2：SQL 反查题

```sql
SELECT A.主键, A.名称
FROM A
WHERE NOT EXISTS (
  SELECT 1
  FROM B
  WHERE B.外键 = A.主键
);
```

或：

```sql
SELECT A.主键, A.名称
FROM A
LEFT JOIN B ON B.外键 = A.主键
WHERE B.外键 IS NULL;
```

采分点：
1. 查询主体是被保留的主表；
2. `NOT EXISTS` 子查询要写关联条件；
3. 左连接反查优先判断连接键为空。

#### 模板3：抢红包 / 首写获胜 2PL 题

```text
T1: XLock(X) → R(X) → 若 X 为空则 W(X) → Commit → Unlock(X)
T2: 等待 T1 释放 X 上的排他锁 → XLock(X) → R(X) → 发现 X 非空 → 不写入 → Commit / Rollback → Unlock(X)
```

采分点：
1. 排他锁保护“读 → 判断 → 写”；
2. 第一个事务先提交并释放锁；
3. 后续事务等待锁释放后再读取；
4. 发现已非空后不再写入。

#### 模板4：上午外围题快速判断模板

- 组成原理题：先看是“速度 / 容量 / 寻址 / I/O 控制”哪一类；
- 数据结构题：先看结构的基本规则，再看操作序列；
- 操作系统题：先看资源分配、调度、同步互斥或地址转换；
- 遇到计算题：先写公式，再代入，避免凭感觉选。

**参考答案：**
1. `T1` 已提交，应 Redo；`T2` 未提交，应 Undo。
2. `X` 应恢复 / 保证为新值 `50`；`Y` 应撤销为旧值 `60`。

**采分点：**
- 区分已提交与未提交事务；
- Redo 使用新值；
- Undo 回旧值。

### 第29题
**参考答案：**
1. `A+ = {A, B, C, D}`，因此候选键为 `A`。
2. 存在传递依赖：`A → B` 且 `B → C`，所以 `A → C` 通过非主属性 `B` 传递得到。
3. 不满足 3NF。可分解为 `R1(A, B, D)` 与 `R2(B, C)`。

**采分点：**
- 求出候选键 `A`；
- 指出 `A → B → C` 的传递依赖；
- 判断不满足 3NF；
- 给出合理分解。

### 第30题
**参考答案：**
- Cache：先写平均访问时间公式，再代入命中率；命中率越高越接近 Cache 时间。
- 死锁：互斥、占有并等待、不可剥夺、循环等待四条件；破坏任一条件即可预防。
- SQL 分组：`SELECT 分组列, 聚合函数 FROM ... GROUP BY 分组列 HAVING 聚合条件`。
- 2PL 首写获胜：`XLock → R → 若空则 W → Commit → Unlock；后者等待释放锁后再读，非空则不写`。
- 日志恢复：已提交写事务 Redo，未提交写事务 Undo，未写数据或检查点前稳定完成通常不处理。

**采分点：**
- 每类模板都要可直接落笔；
- SQL 与 2PL 模板必须体现 Day 30 的失分纠偏；
- 恢复模板必须包含“是否写过数据”。
