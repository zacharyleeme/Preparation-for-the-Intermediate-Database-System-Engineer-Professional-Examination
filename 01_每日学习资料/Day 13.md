### 2.2 为什么 Day 13 要进入 3NF / BCNF 分解策略与综合规范化案例表达

从总览与前几天学习链路看，当前“函数依赖与范式”这一段主线，已经推进到了最关键的收束阶段：

- Day 10：函数依赖与范式设计基础
- Day 11：候选键、属性闭包、范式综合判断
- Day 12：模式分解、无损连接、保持函数依赖
- **Day 13：3NF / BCNF 分解策略与综合规范化案例表达**

也就是说，Day 13 不是另起炉灶，而是把前 3 天已经学过的判断链路真正拧成一条完整的下午题输出链：

> **先算清候选键和最高范式，再看问题依赖，再决定分解策略，再评价分解结果，最后把全过程写成标准答案。**

为什么 Day 13 必须单独做这一层深化？原因有三：

1. **从知识判断到分解策略，还差“选方案”的一步。**  
   Day 12 已经让你知道“无损连接”和“保持函数依赖”分别在看什么，但考试真正拉分的地方，常常不是只问定义，而是追问：
   - 到底该分到 3NF 还是 BCNF？
   - 为什么这样分？
   - 分完后损失了什么，保留了什么？

2. **3NF 与 BCNF 的关系，是规范化综合题的高频陷阱。**  
   很多题会故意设置成：
   - 满足 3NF，但不满足 BCNF；
   - BCNF 分解更严格，但可能不保持函数依赖；
   - 3NF 分解更适合考试场景中的“依赖检查方便 + 无损 + 便于说明”。

3. **Day 12 的真实错题已经证明：你当前更容易丢的是“书写路径分”。**  
   当前最大的短板不是完全不会，而是：
   - 候选键知道个大概，但不写闭包与最小性；
   - 会说“无损”，却不按链路写依据；
   - 模板题知道什么意思，却写不成标准句序。  
   所以 Day 13 的重点，不只是继续学概念，而是要把**规范化综合题的答题动作真正写稳**。

### 2.3 今日核心知识框架

今天要固定下来的，是一套 **“3NF / BCNF 综合规范化题”标准分析框架**：

> **先求候选键与主非主属性 → 判最高范式 → 找出违反高范式的依赖 → 决定分解到 3NF 还是 BCNF → 判无损连接与依赖保持 → 做综合评价。**

#### 1）3NF 和 BCNF 分别想解决什么问题

- **3NF（第三范式）**：重点解决传递依赖、降低冗余，并尽量兼顾“依赖保持 + 无损连接 + 结构可解释”。
- **BCNF（Boyce-Codd Normal Form）**：比 3NF 更严格，要求每个非平凡函数依赖的决定因素都必须是超键，从而进一步减少异常与冗余。

简单记忆：

- **3NF 更像“考试实用型分解”**：通常更容易保留依赖，也更便于写答案；
- **BCNF 更像“结构更纯粹的分解”**：更严格，但有时会牺牲依赖保持性。

#### 2）为什么“满足 3NF”不等于“满足 BCNF”

这是 Day 13 的核心高频考点。

3NF 对某些依赖是“允许放行”的，比如：

- 若依赖 `X → A` 中，`X` 不是超键；
- 但 `A` 是主属性；
- 那么它**可能仍满足 3NF**，但**不满足 BCNF**。

所以判断 BCNF 时，比 3NF 更直接：

> **只要存在某条非平凡函数依赖，其决定因素不是超键，就不满足 BCNF。**

#### 3）综合题里为什么必须先求候选键

因为很多判断都建立在候选键之上：

- 主属性 / 非主属性怎么分，要先看候选键；
- 3NF 的某些例外条件，要先知道右部是不是主属性；
- BCNF 则要看决定因素是不是超键；
- 分解时要知道哪些依赖是真正的问题依赖。

所以综合题只要一上来不先求候选键，后面很容易全部写飘。

#### 4）如何从“问题依赖”过渡到“分解方案”

在考试中，分解不是瞎拆，而是要围绕 **违反高范式的依赖** 去拆。

常见逻辑：

1. 先找出不满足目标范式的依赖；
2. 看这条依赖牵涉哪些属性；
3. 先把这组属性拆成一个子模式；
4. 再把剩余属性与关键连接属性保留下来形成另一个或多个子模式；
5. 最后检查无损连接与依赖保持性。

#### 5）考试里为什么经常先写 3NF，再谈 BCNF

因为 3NF 常常更容易解释清楚：

- 哪些依赖被保留了；
- 为什么这个分解无损；
- 为什么原问题依赖被消除了；
- 为什么结构更合理。

而 BCNF 题目常常进一步追问：

- 更严格的分解是否还保持函数依赖？
- 如果不保持，为什么仍可视为更纯净的设计？
- 如果考试问“更合适的分解”，到底应优先 3NF 还是 BCNF？

因此，做题时通常建议：

> **先把 3NF 的逻辑写稳，再看题目是否明确要求分到 BCNF。**

#### 6）综合评价一个分解方案，至少看哪三个维度

遇到“评价该分解方案是否合理”的题，至少从这三个维度去写：

1. **范式是否提升 / 冗余是否降低**
2. **是否无损连接**
3. **是否保持函数依赖 / 依赖是否便于检查**

若能再补一句“结构更清晰、减少更新异常”，答案会更完整。

#### 7）多表链式分解的无损说明怎么写

Day 12 已经暴露出最大问题之一：

> **把多表分解写成三表总交集判定。**

Day 13 必须纠正为：

- 先看 `R1` 与 `R2` 的公共属性；
- 再看 `R2` 与 `R3` 的公共属性；
- 若是更长链，再继续逐段看；
- 结合依赖链说明“这一步为什么连接不失真”。

也就是说：

> **链式分解按链路逐段解释，不是一次性写总交集公式。**

#### 8）标准书写顺序为什么比“会做”更重要

下午题的采分点往往不是看你是不是“心里知道”，而是看你有没有把关键动作写出来。标准句序最稳的是：

1. 先列原依赖 / 原模式存在的问题
2. 再写候选键或最高范式判断
3. 再给分解结果
4. 再判保持函数依赖
5. 再判无损连接
6. 最后做综合结论

只要句序稳，哪怕内容不是满分，也更容易拿到过程分。

### 2.4 必备结论

- BCNF 比 3NF 更严格，满足 BCNF 的关系模式一定满足 3NF
- 3NF 中允许某些“左部不是超键但右部是主属性”的依赖存在
- BCNF 中只要某条非平凡依赖的左部不是超键，就不能通过
- 3NF 分解通常更容易兼顾无损连接与保持函数依赖
- BCNF 分解更强调结构纯净，但有时可能不保持函数依赖
- 综合题第一步通常不是直接分解，而是先求候选键
- 候选键题必须写“能推出全部属性”和“满足最小性”
- 多表链式分解的无损说明要按相邻子模式逐段写
- 评价分解方案时，至少要写“范式 / 冗余、无损连接、依赖保持”三个维度
- 模板题真正的得分关键，是把关键判断按稳定句序完整写出

### 2.5 高频考点拆解

#### 2.5.1 “满足 3NF 但不满足 BCNF”是高频陷阱

**核心概念：** 3NF 与 BCNF 不是同一个严格度。  
**常见考法：** 给出一组依赖，让你判断“最高范式是 3NF 还是 BCNF”。  
**高频误区：** 认为“只要满足 3NF 就一定满足 BCNF”。  
**快速判断提示：** 先看是否存在“决定因素不是超键”的非平凡依赖。  
**小例子：** `AB → C`、`C → B` 中，若候选键为 `AB` 和 `AC`，则 `C → B` 可能让它满足 3NF 但不满足 BCNF。

#### 2.5.2 综合题第一步一定是候选键

**核心概念：** 候选键是主属性、3NF 例外条件、BCNF 判定的共同前提。  
**常见考法：** 先求候选键，再判断最高范式，再给分解方案。  
**高频误区：** 一上来直接猜“这是 2NF / 3NF / BCNF”。  
**快速判断提示：** 先用闭包法看哪个属性组能推出全部属性，再判断最小性。  
**小例子：** 若 `A → B`、`B → C`、`D` 无法被推出，则先试 `AD` 的闭包，而不是直接猜键。

#### 2.5.3 3NF 分解更常被拿来写“考试答案”

**核心概念：** 3NF 分解通常更便于解释依赖保持。  
**常见考法：** 问“为什么该分解更合理”“为何优先分到 3NF”。  
**高频误区：** 以为考试里一定追求 BCNF。  
**快速判断提示：** 若题目强调“依赖便于检查”“原依赖尽量保留”，先想 3NF。  
**小例子：** 某 BCNF 分解虽然更严格，但必须跨表才能验证原依赖；而 3NF 分解可能更适合答题场景。

#### 2.5.4 多表无损说明不能偷换成总交集

**核心概念：** 链式分解的说明逻辑是“逐段连接”。  
**常见考法：** 给出 `R1、R2、R3`，要求说明是否可视为合理无损分解。  
**高频误区：** 直接写 `R1 ∩ R2 ∩ R3 → Ri`。  
**快速判断提示：** 按 `R1∩R2`、`R2∩R3` 分别说明，再补“沿依赖链展开”。  
**小例子：** `R1(A,B)`、`R2(B,C)`、`R3(C,D)` 应先写 B，再写 C，而不是写三表总交集。

#### 2.5.5 模板题考的不是“意思差不多”，而是“步骤完整”

**核心概念：** 下午题按采分点给分，关键步骤没写就会掉分。  
**常见考法：** 要求写 4～6 句标准答案或综合评价。  
**高频误区：** 只写最后结论，不写候选键、问题依赖、无损 / 保持依赖依据。  
**快速判断提示：** 强制按“原依赖 / 问题 → 分解结果 → 依赖保持 → 无损 → 结论”落笔。  
**小例子：** “该分解更合理”这句话必须前面有依据支撑，不能直接空降。

### 2.6 下午题视角下的题型映射

| 下午题类型 | 常见问法 | 作答关键动作 | 常见丢分点 |
|---|---|---|---|
| 3NF / BCNF 判断题 | “判断该关系模式最高达到几范式” | 先求候选键，再看问题依赖，最后给结论 | 直接猜范式，不写候选键与判断过程 |
| 规范化分解题 | “将关系模式分解到 3NF / BCNF” | 先指出违反目标范式的依赖，再给分解方案 | 只给分解结果，不说明为何这样拆 |
| 分解方案评价题 | “说明该分解为何合理” | 同时写范式提升、无损连接、依赖保持 | 只写一个维度，缺少综合评价 |

### 2.7 标准答题模板 / 采分点提示

#### 模板1：3NF / BCNF 综合判断题
1. 先求候选键，并区分主属性 / 非主属性。  
2. 再判断是否存在部分依赖、传递依赖或“决定因素不是超键”的依赖。  
3. 若存在违反 BCNF 的依赖，说明是否仍满足 3NF。  
4. 最后给出“最高达到几范式”的结论。

#### 模板2：规范化分解题
1. 先写原关系模式存在的具体问题依赖。  
2. 再写准备分解出的子关系模式。  
3. 接着说明原依赖在哪些子模式中被保留。  
4. 再说明该分解是否无损连接。  
5. 最后写“分解后结构更清晰 / 冗余降低 / 异常减少”的总结。

#### 模板3：综合评价 / 标准答案复写题
1. 先列原依赖或原模式问题。  
2. 再写候选键或最高范式结论。  
3. 再写分解结果。  
4. 再判保持函数依赖。  
5. 再判无损连接。  
6. 最后做“更合理 / 更规范 / 但可能存在某些代价”的综合评价。

- 写出依赖推出链路
- 说明最小性

**作答提醒：**
- 先写候选键结论
- 再写推出路径
- 最后补“满足最小性”

### 第20题
**参考答案：**  
该关系模式最高只能达到 **1NF**。因为候选键为 `(学号, 课程号)`，而非主属性 `教师号` 只依赖于 `课程号`，属于对组合键的部分函数依赖；同时 `教师号 → 教师姓名` 还体现出传递依赖。因此它不能达到 2NF，更不可能达到 3NF / BCNF。

**采分点：**
- 写出最高范式为 1NF
- 指出 `课程号 → 教师号` 导致部分依赖
- 能提及 `教师号 → 教师姓名` 的传递依赖问题

**作答提醒：**
- 先写候选键
- 再指出部分依赖
- 再补充传递依赖更完整

### 第21题
**参考答案：**  
该分解可以视为合理的 3NF 分解。因为：
1. `R1(学号, 课程号, 成绩)` 保留了 `(学号, 课程号) → 成绩`；
2. `R2(课程号, 教师号)` 保留了 `课程号 → 教师号`；
3. `R3(教师号, 教师姓名)` 保留了 `教师号 → 教师姓名`；
4. 原有关键依赖均能在对应子模式中表达，且分解沿依赖链展开，可视为合理无损，并能减少冗余与异常。

**采分点：**
- 给出“合理的 3NF 分解”结论
- 逐条对照原依赖与子模式
- 能写出“保持依赖 + 可视为合理无损 + 减少冗余”中的至少两项

**作答提醒：**
- 不要只写“分解后更合理”
- 至少写出依赖保留与无损 / 冗余改善中的关键依据

### 第22题
**参考答案：**  
原关系模式的候选键为 `(学号, 课程号)`，其中 `课程号 → 教师号` 和 `教师号 → 教师姓名` 使原模式存在部分依赖与传递依赖问题。将其分解为 `R1(学号, 课程号, 成绩)`、`R2(课程号, 教师号)`、`R3(教师号, 教师姓名)` 后，三条关键函数依赖分别保留在对应子模式中，因此该分解保持函数依赖。进一步看，分解沿原依赖链逐步展开，连接过程有明确依据，可视为合理的无损分解。综上，该分解既提升了规范化程度，又降低了冗余和更新异常。

**采分点：**
- 顺序完整：候选键 / 问题依赖 → 分解结果 → 保持依赖 → 无损 → 结论
- 句子结构清楚
- 至少写出两个判断结论及依据

**作答提醒：**
- 先写问题，再写分解
- 结论前要有依据

### 第23题
**参考答案：**  
候选键为 **AB** 和 **AC**。因为：
- `AB → C`，所以 `AB⁺ = {A, B, C}`，故 `AB` 能决定全部属性；
- 由 `C → B` 可得 `AC → B`，因此 `AC⁺ = {A, C, B}`，故 `AC` 也能决定全部属性；
- `A`、`B`、`C` 单独都不能决定全部属性，因此 `AB` 和 `AC` 是候选键。

**采分点：**
- 写出候选键 `AB`、`AC`
- 能写出闭包或推出路径
- 说明最小性

**作答提醒：**
- 重点别漏第二个候选键 `AC`
- 闭包写法最稳

### 第24题
**参考答案：**  
该关系模式最高达到 **3NF，但不满足 BCNF**。因为候选键为 `AB` 和 `AC`，所以 `B`、`C` 都是主属性。依赖 `C → B` 中，`C` 不是超键，因此不能满足 BCNF；但右部 `B` 是主属性，因此该依赖在 3NF 口径下仍可被接受，所以该模式满足 3NF。

**采分点：**
- 给出“3NF 但不满足 BCNF”的结论
- 指出 `C` 不是超键
- 指出 `B` 是主属性，因此 3NF 仍可通过

**作答提醒：**
- 先写候选键
- 再写 `C → B` 的双重判断

### 第25题
**参考答案：**  
该分解 **无损连接，但不保持原函数依赖**。因为 `R1(C, B)` 与 `R2(A, C)` 的公共属性是 `C`，且 `C → B`，所以公共属性 `C` 能决定 `R1` 的全部属性，可据此判断该二元分解无损连接。  
但原依赖 `AB → C` 在分解后没有直接保留在某个子模式中，也难以仅由投影依赖直接推出，因此该分解不保持原函数依赖。

**采分点：**
- 明确写出“无损连接”结论
- 能写出公共属性 `C` 及 `C → B` 的依据
- 明确写出“不保持 `AB → C`”

**作答提醒：**
- 这题要把“无损”和“保持依赖”拆开写
- 不要因为无损就顺手写成保持依赖

### 第26题
**参考答案：**  
BCNF 更严格，因为它要求每个非平凡函数依赖的决定因素都必须是超键，因此能进一步减少冗余与异常。  
但考试中常常先讲 3NF，是因为 3NF 分解通常更便于保持函数依赖，也更容易把“为什么这样分”写清楚。  
因此，若题目强调结构更纯粹，可以继续分析 BCNF；若题目强调依赖保持与答题可解释性，3NF 往往是更稳的第一步。

**采分点：**
- 写出 BCNF 更严格
- 写出 3NF 更便于依赖保持 / 答题说明
- 能写出“考试中常先讲 3NF”的原因

**作答提醒：**
- 不要只写“BCNF 更高级”这种空话
- 要说明“更严格”和“更好写答案”的差别

### 第27题
**参考答案：**  
候选键为 **A**。因为：
- `A → B`
- `B → C`
- `C → D`
所以 `A⁺ = {A, B, C, D}`，A 能推出全部属性。又因为 A 为单属性，无法再继续删减，因此 A 是最小超键，也就是候选键。

**采分点：**
- 写出候选键 A
- 写出闭包 `A⁺`
- 说明最小性

**作答提醒：**
- 一定要写 `A⁺`
- 别把“能推出全部属性”和“最小性”分开丢掉其中一个

### 第28题
**参考答案：**  
该分解保持原函数依赖。因为：
- `A → B` 保留在 `R1(A, B)` 中；
- `B → C` 保留在 `R2(B, C)` 中；
- `C → D` 保留在 `R3(C, D)` 中。  
原模式中的关键依赖都能在分解后的对应子模式中直接检查。

**采分点：**
- 给出“保持函数依赖”的结论
- 逐条列出三条依赖在对应子模式中保留
- 用语清楚

**作答提醒：**
- 最稳写法就是逐条对照
- 不要只写“依赖都保留了”

### 第29题
**参考答案：**  
该分解可以视为合理的无损分解。分析思路可写为：
1. `R1` 与 `R2` 的公共属性为 `B`，且 `B → C`，因此这一段连接有明确依赖依据；
2. `R2` 与 `R3` 的公共属性为 `C`，且 `C → D`，因此后一段连接同样有依据；
3. 该分解沿 `A → B → C → D` 的依赖链逐段展开，连接时不会无依据地产生伪组合，因此可视为合理无损分解。

**采分点：**
- 给出“合理无损”的结论
- 写出两段公共属性 `B`、`C`
- 能结合依赖链说明“逐段连接”思路

**作答提醒：**
- 不要写三表总交集
- 先写第一段，再写第二段

### 第30题
**参考答案：**  
原关系模式的主要函数依赖为 `A → B`、`B → C`、`C → D`，其中 A 的闭包可推出全部属性，因此候选键为 A。将其分解为 `R1(A, B)`、`R2(B, C)`、`R3(C, D)` 后，这三条依赖分别保留在对应子模式中，因此该分解保持函数依赖。进一步看，`R1` 与 `R2` 通过公共属性 B 连接，`R2` 与 `R3` 通过公共属性 C 连接，分解沿依赖链逐段展开，可视为合理的无损分解。综上，该分解既保留了原依赖，也使结构更清晰。

**采分点：**
- 顺序完整：原依赖 → 候选键 → 分解结果 → 保持依赖 → 无损连接 → 结论

- 句子结构清楚

**作答提醒：**

- 把“保持依赖”和“无损连接”分开写更稳
