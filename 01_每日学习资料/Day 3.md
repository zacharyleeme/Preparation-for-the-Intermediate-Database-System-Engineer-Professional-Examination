### 2.3 今日核心知识框架

SQL 查询最基础的主干结构是：

```sql
SELECT 列名
FROM 表名
WHERE 条件;
```

可以先用一句人话理解：

- `FROM`：从哪张表取数据
- `WHERE`：筛哪些记录
- `SELECT`：最终显示哪些字段

虽然书写时通常 `SELECT` 写在最前面，但思考时更推荐按这个顺序理解：

1. 从哪张表取数据
2. 需要哪些条件筛选
3. 最后输出哪些列

这和 Day 2 的关系代数思维完全一致：

- 选择（σ）≈ `WHERE`
- 投影（π）≈ `SELECT 列名`

### 2.4 必备结论

- `SELECT *` 表示查询表中的所有列
- `SELECT 列名1, 列名2` 表示只查询指定字段
- `FROM 表名` 指定数据来源
- `WHERE` 用于按条件筛选记录
- `DISTINCT` 用于去除重复值
- `ORDER BY` 用于排序
- `ASC` 表示升序，`DESC` 表示降序
- SQL 中：
  - **筛行**主要看 `WHERE`
  - **取列**主要看 `SELECT`
- `WHERE` 常见比较运算：
  - `=`
  - `<>` 或 `!=`
  - `>`
  - `<`
  - `>=`
  - `<=`
- 常见逻辑运算：
  - `AND`：并且
  - `OR`：或者
  - `NOT`：非
- 区间判断常见写法：
  - `BETWEEN ... AND ...`
- 集合判断常见写法：
  - `IN (...)`
- 模糊查询常见写法：
  - `LIKE`
  - `%` 表示任意长度字符
  - `_` 表示单个字符

### 2.5 高频考点拆解

#### 2.5.1 `SELECT` 与 `SELECT *`

设有学生表：

- Student(学号, 姓名, 性别, 年龄, 专业)

如果要查询整张表所有字段，可写：

```sql
SELECT *
FROM Student;
```

如果只想看姓名和专业，则写：

```sql
SELECT 姓名, 专业
FROM Student;
```

高频误区：

- `SELECT *` 不是“查所有记录”的意思，而是“查所有列”
- 是否筛选记录，要看有没有 `WHERE`

#### 2.5.2 `WHERE`：按条件筛选记录

如果要求“查询年龄大于 20 岁的学生信息”，可写：

```sql
SELECT *
FROM Student
WHERE 年龄 > 20;
```

如果要求“查询年龄大于 20 岁学生的姓名”，可写：

```sql
SELECT 姓名
FROM Student
WHERE 年龄 > 20;
```

和关系代数对应：

- `σ 年龄>20 (Student)`：先筛出年龄大于 20 的记录
- `π 姓名 (σ 年龄>20 (Student))`：再取姓名列

因此要始终记住：

- `WHERE` 负责筛行
- `SELECT` 负责取列

#### 2.5.3 `DISTINCT`：去重

如果 Student 表中“专业”字段有重复值，要求查询“有哪些专业”，应该去重：

```sql
SELECT DISTINCT 专业
FROM Student;
```

如果不加 `DISTINCT`，重复的专业会重复显示。

高频误区：

- `DISTINCT` 是对查询结果去重
- 它不是删除原表中的重复数据
- 它通常和字段查询一起考察，不要和排序、筛选混淆

#### 2.5.4 `ORDER BY`：排序

如果要求“按年龄从大到小查看学生信息”：

```sql
SELECT *
FROM Student
ORDER BY 年龄 DESC;
```

如果要求“按年龄从小到大排序”：

```sql
SELECT *
FROM Student
ORDER BY 年龄 ASC;
```

注意：

- 默认通常按升序理解
- 题目更喜欢考 `DESC` 的含义：降序

#### 2.5.5 多条件查询

如果要求“查询计算机专业且年龄大于 20 岁的学生姓名”：

```sql
SELECT 姓名
FROM Student
WHERE 专业 = '计算机' AND 年龄 > 20;
```

如果要求“查询计算机专业或软件工程专业的学生姓名”：

```sql
SELECT 姓名
FROM Student
WHERE 专业 = '计算机' OR 专业 = '软件工程';
```

常考点：

- `AND`：条件都要满足
- `OR`：满足其一即可
- 多条件题不要只看局部，要看逻辑关系

#### 2.5.6 `IN`、`BETWEEN`、`LIKE`

1. `IN`

查询专业是计算机或软件工程的学生：

```sql
SELECT 姓名
FROM Student
WHERE 专业 IN ('计算机', '软件工程');
```

2. `BETWEEN ... AND ...`

查询年龄在 18 到 22 岁之间的学生：

```sql
SELECT 姓名
FROM Student
WHERE 年龄 BETWEEN 18 AND 22;
```

3. `LIKE`

查询姓名以“张”开头的学生：

```sql
SELECT 姓名
FROM Student
WHERE 姓名 LIKE '张%';
```

查询姓名为两个字且姓“张”的学生：

```sql
SELECT 姓名
FROM Student
WHERE 姓名 LIKE '张_';
```

易错点：

- `%` 是任意长度字符
- `_` 是单个字符
- `BETWEEN a AND b` 一般表示包含边界

### 2.6 SQL 与关系代数的对应直觉

今天不要求把所有 SQL 都翻译成复杂关系代数，但至少要有这个映射意识：

- `FROM Student`  
  对应：先确定基础关系 Student
- `WHERE 年龄 > 20`  
  对应：选择 `σ 年龄>20`
- `SELECT 姓名`  
  对应：投影 `π 姓名`

例如：

```sql
SELECT 姓名
FROM Student
WHERE 年龄 > 20;
```

本质上就是：

```text
π 姓名 (σ 年龄>20 (Student))
```

这正是 Day 2 到 Day 3 的衔接主线。

### 2.7 易混点速记卡片

- **SELECT \***  
  - 查所有列
  - 不是查“所有满足条件以外的全部内容”
  - 关键词：星号 = 所有字段

- **SELECT vs WHERE**
  - SELECT：决定显示哪些列
  - WHERE：决定保留哪些行
  - 关键词：取列 / 筛行

- **DISTINCT**
  - 查询结果去重
  - 不修改原表
  - 关键词：结果去重，不是删数据

- **AND vs OR**
  - AND：同时满足
  - OR：满足其一
  - 关键词：交集 / 并集直觉

- **LIKE**
  - `%`：任意长度
  - `_`：单个字符
  - 关键词：模糊匹配

- **ORDER BY**
  - ASC：升序
  - DESC：降序
  - 关键词：排序方向题高频出现
