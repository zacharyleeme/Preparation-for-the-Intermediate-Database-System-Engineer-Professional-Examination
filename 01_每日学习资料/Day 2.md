### 2.2 为什么 Day 2 学关系代数

Day 1 已经建立了数据库系统基础和关系模型术语。接下来最自然、最高性价比的衔接，就是进入**关系代数**：

- 它是关系数据库查询思想的“底层语言”；
- 它和 SQL 高度相关，很多 SQL 题本质上都能还原成关系代数题；
- 上午选择题经常考“某运算的含义”“表达式结果”“运算适用条件”；
- 下午题中的查询设计、关系模式分析，也依赖这种“按步骤构造结果集”的思维。

简单说：**关系代数学明白，SQL 会轻松很多。**

### 2.3 必备结论

- 关系代数是以关系为运算对象、以关系为运算结果的一种形式化查询语言。
- 关系代数中的基本运算通常包括：
  - 选择（σ）
  - 投影（π）
  - 并（∪）
  - 差（−）
  - 笛卡尔积（×）
- 常见扩展运算包括：
  - 交（∩）
  - 连接（⨝）
  - 自然连接
  - 除（÷）

- **选择（σ）**：从关系中选出满足条件的若干**行**。
- **投影（π）**：从关系中取出需要的若干**列**。
- **选择管行，投影管列**，这是最常考、也最容易混淆的一组。

- 并、差、交运算要求两个关系**并相容**：
  - 属性个数相同
  - 对应属性的域相同或类型兼容
- 笛卡尔积会得到两个关系所有元组的两两组合，结果元组数 = 两边元组数相乘。
- 连接本质上是“先笛卡尔积，再按条件筛选”。
- 自然连接是在等值连接基础上，自动去掉重复属性列的一种连接。
- 关系代数运算结果仍然是关系，因此可以继续嵌套运算。
- 关系代数表达式常见处理顺序：
  1. 明确基础关系
  2. 判断是否需要筛行（选择）
  3. 判断是否需要多表组合（连接）
  4. 判断最终保留哪些列（投影）

### 2.4 高频考点拆解

#### 2.4.1 选择（σ）与投影（π）

- 选择：按条件挑记录
- 投影：按字段取列

例：
- Student(学号, 姓名, 性别, 年龄, 专业)

若要求“查询年龄大于 20 岁的学生姓名”：

1. 先选出年龄 > 20 的元组  
   `σ 年龄>20 (Student)`

2. 再投影出姓名列  
   `π 姓名 (σ 年龄>20 (Student))`

记忆口诀：
- **条件写在 σ 后面**
- **字段写在 π 后面**

#### 2.4.2 并、差、交

设：
- A(学号, 姓名)
- B(学号, 姓名)

则：
- `A ∪ B`：属于 A 或属于 B 的元组
- `A − B`：属于 A 但不属于 B 的元组
- `A ∩ B`：同时属于 A 和 B 的元组

考试常问：
- 哪些运算要求并相容？
  - 并、差、交
- 哪些不要求并相容？
  - 选择、投影、笛卡尔积、连接

#### 2.4.3 笛卡尔积（×）

设：
- A 有 m 个元组
- B 有 n 个元组

则：
- `A × B` 的元组数为 `m × n`

这是高频基础计算题。

但在真实查询里，笛卡尔积本身通常不是最终目标，更多是连接的中间逻辑基础。

#### 2.4.4 连接（⨝）

连接就是把两个关系中“有联系的元组”拼起来。

例如：
- Student(学号, 姓名, 专业号)
- Major(专业号, 专业名)

若要求“查询每个学生的姓名及其专业名称”，则要按 `专业号` 连接两张表。

关系代数思想：
1. 先把两表关联起来
2. 再保留需要字段

可写为：
`π 姓名, 专业名 (Student ⨝ Student.专业号 = Major.专业号 Major)`

如果使用自然连接，要求两个关系中存在同名且语义相同的连接属性。

#### 2.4.5 自然连接 vs 等值连接

- **等值连接**：按相等条件连接，结果中通常保留两边的连接属性
- **自然连接**：在等值连接基础上，自动去掉重复的同名连接列

高频误区：
- 不是“属性值相等就一定能自然连接”
- 需要两边有**同名且可对应**的属性，且语义一致

### 2.5 关系代数到 SQL 的初步对应

这是 Day 2 先建立直觉，不要求今天全部吃透 SQL 细节。

- 选择（σ）≈ `WHERE`
- 投影（π）≈ `SELECT 字段`
- 连接（⨝）≈ `JOIN ... ON ...`
- 并（∪）≈ `UNION`
- 差（−）≈ SQL 中可借助 `EXCEPT`（不同数据库实现略有差异）
- 交（∩）≈ SQL 中可借助 `INTERSECT`（不同数据库实现略有差异）

考试思路不要死记符号，要抓本质：
- **σ 是筛行**
- **π 是取列**
- **⨝ 是拼表**
- **∪ / − / ∩ 是集合运算**

### 2.6 易混点速记卡片

- **选择 vs 投影**
  - 选择：处理行
  - 投影：处理列
  - 误判点：把“条件”写到投影里
  - 关键词：选行投列

- **并相容**
  - 要求：属性个数对应相同、类型兼容
  - 误判点：只看“列数一样”就认为可以并
  - 关键词：结构要对齐

- **笛卡尔积 vs 连接**
  - 笛卡尔积：全组合
  - 连接：有条件的组合
  - 误判点：把连接结果元组数直接按乘法计算
  - 关键词：连接 = 积 + 筛

- **等值连接 vs 自然连接**
  - 等值连接：按相等条件连
  - 自然连接：按同名公共属性连，并去重重复列
  - 误判点：以为二者完全等同
  - 关键词：自然连接会“自动去重复属性列”

- **关系代数表达式书写顺序**
  - 实际思考：先从表出发，再筛行，再连接，再取列
  - 误判点：一上来就直接写最终投影，漏掉中间条件
  - 关键词：先找数据来源，再做运算
