# Day 1｜2026-04-05｜距离考试 48 天

## 1. 今日学习目标

1. 建立数据库系统工程师考试的第一层知识骨架。
2. 吃透数据库系统、数据库、数据库管理系统（DBMS）的区别。
3. 掌握三级模式两级映像与数据独立性，这是高频选择题考点。
4. 初步掌握关系模型中的基本概念：属性、域、元组、码、关系模式。
5. 完成 30 道围绕基础概念与关系模型的练习题，为后续 SQL、范式、设计题打底。

## 2. 学习内容

### 2.1 必备结论

- 数据（Data）是描述事物的符号记录；数据库（DB）是长期存储在计算机内、有组织、可共享的数据集合。
- 数据库管理系统（DBMS）是位于用户和操作系统之间的数据管理软件，负责定义、操纵、控制和维护数据库。
- 数据库系统（DBS） = 数据库 + 数据库管理系统 + 应用程序 + 数据库管理员 + 用户 + 硬件/软件环境。
- 数据库技术的核心目标：实现数据共享、减少冗余、提高一致性、保证安全性与完整性。
- 三级模式：外模式、模式、内模式。
- 两级映像：外模式/模式映像、模式/内模式映像。
- 外模式/模式映像保证逻辑独立性：概念模式变化时，外模式可相对稳定。
- 模式/内模式映像保证物理独立性：存储结构变化时，概念模式可相对稳定。
- 关系模型中，关系就是一张二维表，但考试中要按严格术语理解，不等于“随便一张 Excel 表”。
- 关系的列称属性，行称元组；属性取值范围称域。
- 候选码：能唯一标识元组的最小属性组；主码：从候选码中选定的一个。
- 主属性：包含在任一候选码中的属性；非主属性：不包含在任何候选码中的属性。
- 关系中元组次序无关、属性次序原则上也无关。
- 关系中不允许出现完全相同的元组。
- 分量必须取原子值，这是关系模型“规范化/第一范式思想”的基础。
- 常考判断：数据库系统强调“结构化数据管理”，文件系统难以充分解决数据冗余、一致性和共享控制问题。

### 2.2 易混点速记卡片

- **数据库 vs 数据库系统**
  - 数据库：被管理的数据集合
  - 数据库系统：数据库 + DBMS + 人员 + 应用 + 环境
  - 误判点：把 DB 和 DBS 混为一谈
  - 关键词：系统一定包含“管理”和“运行主体”

- **DBMS vs 操作系统**
  - DBMS：管理数据逻辑与存取控制
  - 操作系统：管理计算机硬件与基础资源
  - 误判点：把文件管理功能等同于数据库管理功能
  - 关键词：DBMS 面向“数据组织与控制”

- **外模式 / 模式 / 内模式**
  - 外模式：用户视图、子模式
  - 模式：全局逻辑结构
  - 内模式：物理存储结构
  - 误判点：把模式当作物理结构
  - 关键词：外-用户；模-逻辑；内-存储

- **逻辑独立性 vs 物理独立性**
  - 逻辑独立性：概念模式变化，不影响外模式/应用
  - 物理独立性：内模式变化，不影响概念模式
  - 误判点：把二者方向搞反
  - 关键词：逻辑看“外—模”；物理看“模—内”

- **属性 vs 域 vs 元组**
  - 属性：列名/字段
  - 域：属性允许取值的集合
  - 元组：一行记录
  - 误判点：把域理解成“某一列当前的所有值”
  - 关键词：域是“取值范围”，不是“已取的值”

- **候选码 vs 主码**
  - 候选码：满足唯一标识且最小
  - 主码：选中的那个候选码
  - 误判点：认为一个关系只能有一个候选码
  - 关键词：候选码可多个，主码只选一个

- **主属性 vs 非主属性**
  - 主属性：出现在任一候选码中的属性
  - 非主属性：不属于任何候选码
  - 误判点：把“主码中的属性”误解为“主属性的全部定义”
  - 关键词：任一候选码即可
