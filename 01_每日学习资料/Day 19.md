### 2.3 今日核心知识框架

今天要固定下来的，是一套 **“数据库保障类题目标准分析框架”**：

> **先看业务目标与风险 → 再定安全 / 备份 / 运维手段 → 再写为什么有效 → 最后补代价、边界与结论。**

#### 1）安全题先看“保护什么、怕什么”，再谈控制手段

安全设计题的第一反应，不应该是直接写“加密、审计、备份”。

更稳妥的起手式是先判断：
- 保护对象是什么：个人隐私、交易数据、核心业务库、备份文件；
- 主要风险是什么：越权访问、数据泄露、误删误改、恶意篡改、口令暴力尝试；
- 最关键的安全目标是什么：保密性、完整性、可用性、可审计性。

所以安全题真正的起手顺序应是：

> **先识别资产与风险，再决定权限、视图、审计、加密等控制手段。**

#### 2）权限控制题考的不是“有没有授权”，而是“授权是否分层、是否最小化”

权限控制题里最常见的高频动作包括：
- 用**角色**统一管理权限；
- 用**视图**限制可见列或可见行；
- 给分析人员、业务人员、管理员分别授予不同级别权限；
- 对关键操作保留审计痕迹；
- 遵循**最小权限原则**与必要分离原则。

因此，看到“只能看部分数据、不能改、不能看敏感列”的题干时，不能只写“授予 SELECT 权限”，而要想到：

> **角色 + 视图 + 只读权限 + 审计** 这一整套分层控制。

#### 3）备份题先定恢复目标，而不是先背备份种类

备份恢复题最容易犯的错误，是一看到“全量 / 增量 / 差异”就开始直接选方案。

但下午题真正更想考的是：
- 业务允许丢多少数据？
- 系统允许停多久？
- 是否需要异地保存？
- 是否需要演练验证？
- 是否要兼顾加密与存储成本？

也就是：

> **先确定 RPO / RTO，再反推备份策略。**

#### 4）RPO 与 RTO 是恢复题里的关键判断口径

- **RPO（Recovery Point Objective）**：最多能容忍丢多少数据；
- **RTO（Recovery Time Objective）**：最多能容忍停多久。

如果题目说：
- 最多只能丢 15 分钟数据；
- 最多只能停 2 小时；

那答案里就要明确：
- 为什么需要更频繁的日志 / 增量类保护；
- 为什么恢复步骤不能太复杂；
- 为什么仅有偶尔一次全量备份往往不够。

#### 5）备份不等于高可用，恢复也不等于切换

这是下午题很容易混淆的一组概念：
- **备份** 更偏向“数据能不能找回来”；
- **高可用 / 主备 / 冗余切换** 更偏向“服务能不能尽快继续提供”。

所以如果题目问“为什么有备份还不够”，要想到：

> **备份解决的是恢复数据，高可用解决的是缩短业务中断时间，两者不能互相替代。**

#### 6）恢复演练与异地备份是“方案可用性”常见采分点

很多人会写：
- 要做全量备份；
- 要做增量备份；
- 要做日志备份；

但漏掉两件真正很重要的事：
- **异地 / 离线保存**：防止本地硬件故障、误删、勒索、机房事故一起把备份带走；
- **恢复演练**：验证备份是否真的可恢复，而不是“看上去有文件”。

这两层经常是区分“会背方案”和“会写完整答案”的关键。

#### 7）运维分析题要把监控、告警、审计、容量、慢查询放在一起看

运维分析题往往不会只给你一个单点问题，而会混合给出：
- 失败登录激增；
- 磁盘空间告急；
- 慢 SQL 增多；
- 备份任务失败；
- CPU / I/O 波动。

这类题的关键不在于分别定义这些现象，而在于：

> **先做优先级判断，再分析它们之间是否存在关联，再给出应急动作与长期改进方案。**

#### 8）综合题真正考的是“能不能把手段写成协同关系”

数据库保障类综合题里，常见手段包括：
- 角色 / 权限 / 视图；
- 审计 / 告警 / 日志；
- 备份 / 恢复 / 演练；
- 加密 / 异地保存；
- 容量监控 / 性能监控 / 高可用措施。

高分答案不是把这些名词堆在一起，而是写清：
- 哪个手段解决什么问题；
- 为什么要配合使用；
- 这样做的收益是什么；
- 代价和管理复杂度是什么。

所以今天的核心，是把数据库保障类题目也写成一套稳定套路：

> **目标、风险、手段、原因、验证、代价、结论。**

### 2.4 必备结论

- 数据库安全设计通常先识别保护对象与风险，再选择控制手段
- 最小权限原则要求用户只拥有完成当前工作所必需的最小权限
- 角色适合统一授权管理，视图适合控制可见范围与敏感列暴露
- 审计的核心价值之一，是让关键操作、失败登录、权限变更可追踪、可回溯
- RPO 关注可容忍的数据丢失窗口，RTO 关注可容忍的服务中断时间
- 备份策略应围绕恢复目标设计，而不是孤立背诵全量 / 增量 / 差异的定义
- 备份不等于高可用，数据恢复能力与服务连续性能力不能互相替代
- 异地备份与恢复演练是检验方案“是否真正可用”的高频采分点
- 运维分析题往往要同时看安全事件、性能告警、容量风险与备份状态
- 综合题不能只写手段，还要写各自作用、协同原因与管理代价

### 2.5 高频考点拆解

#### 2.5.1 最小权限原则是权限设计题的起手词

**核心概念：** 用户只应获得完成任务所必需的最小权限。  
**常见考法：** “分析人员只能看汇总，不能改表、不能看敏感列，如何设计？”  
**高频误区：** 只写“授予查询权限”，不写角色分层与可见范围控制。  
**快速判断提示：** 只要题目出现“只能看”“不能改”“不能看敏感信息”，就先想到最小权限。  
**小例子：** 财务分析员只看部门汇总数据，不应直接拥有基础明细表的全部查询权。

#### 2.5.2 角色与视图经常成对出现

**核心概念：** 角色解决“谁有什么权限”，视图解决“能看到什么数据”。  
**常见考法：** “为什么不能直接给业务用户表级 SELECT？”  
**高频误区：** 只写角色，不写视图；或只写视图，不写只读权限与审计。  
**快速判断提示：** 涉及敏感列、敏感行、只看汇总时，视图往往是关键。  
**小例子：** 给 `sales_report_role` 授权查询销售汇总视图，而不是直接查询工资明细表。

#### 2.5.3 RPO / RTO 决定备份策略强度

**核心概念：** RPO 决定数据丢失容忍度，RTO 决定恢复速度要求。  
**常见考法：** 给业务中断容忍条件，让你选备份与恢复方案。  
**高频误区：** 把 RPO 和 RTO 混成一件事，只写“要多备份”。  
**快速判断提示：** 题目里只要出现“最多丢多少数据、最多停多久”，就一定要把 RPO / RTO 明确写出来。  
**小例子：** “最多丢 15 分钟数据”意味着不能只靠每周一次全量备份。

#### 2.5.4 异地备份与恢复演练是完整方案常见漏点

**核心概念：** 有备份文件，不等于真实可恢复。  
**常见考法：** “为什么还要做异地保存 / 恢复演练？”  
**高频误区：** 只列备份种类，不写验证与保存策略。  
**快速判断提示：** 只要题目是“方案设计题”，就要警惕自己是否漏了异地与演练。  
**小例子：** 备份都放在同一台服务器或同一磁盘上，本地故障可能把生产库和备份一起带走。

#### 2.5.5 备份与高可用解决的是不同问题

**核心概念：** 备份偏数据恢复，高可用偏服务连续。  
**常见考法：** “为什么有备份还不够？” “为什么还需要主备或冗余？”  
**高频误区：** 误以为“只要定期备份，就自然满足低中断要求”。  
**快速判断提示：** 题目涉及“业务连续性”“快速切换”“停机容忍度”时，要主动区分这两个概念。  
**小例子：** 备份可能让你恢复昨天的数据，但不一定能让系统在几分钟内继续提供服务。

#### 2.5.6 运维分析题先做优先级，再做关联判断

**核心概念：** 失败登录、慢 SQL、磁盘告急、备份失败等现象可能是同时发生、互相影响的。  
**常见考法：** 给一组告警，让你判断先处理什么、为什么、后续如何加固。  
**高频误区：** 把每个现象拆开背定义，不写先后顺序与联动措施。  
**快速判断提示：** 一看到多告警并发，就先写“识别业务风险最高项，再排查关联”。  
**小例子：** 失败登录激增可能是安全事件，磁盘接近满可能又会导致备份失败和数据库性能抖动。

### 2.6 下午题视角下的题型映射

| 下午题类型 | 常见问法 | 作答关键动作 | 常见丢分点 |
|---|---|---|---|
| 安全控制设计题 | “如何保障敏感数据不被越权查看或修改” | 先写保护对象与风险，再写角色 / 视图 / 权限 / 审计 | 只写加密，不写最小权限与审计 |
| 权限分层题 | “为什么不能直接授予表级查询权限” | 先写角色，再写视图，再写只读与审计 | 只写角色，不写可见范围控制 |
| 备份恢复方案题 | “在给定恢复目标下如何设计备份策略” | 先写 RPO / RTO，再定全量 / 增量 / 日志 / 异地 / 演练 | 直接列备份方式，不写恢复目标 |
| 运维分析题 | “多类告警同时出现时如何判断与处理” | 先做优先级判断，再看告警关联，再给应急与长期措施 | 只逐条解释告警，不写先后顺序 |
| 综合保障题 | “安全、备份、审计、监控如何协同保障数据库” | 分别写作用，再写为什么合起来有效，最后补代价 | 只列手段，不写协同关系与结论 |

### 2.7 标准答题模板 / 采分点提示

#### 模板1：安全设计题
1. 先写保护对象是什么、主要风险是什么。  
2. 再写安全目标：保密性、完整性、可用性或可审计性。  
3. 接着写控制手段：角色、最小权限、视图、审计、加密等。  
4. 然后说明这些手段分别如何降低越权访问、误删误改或泄露风险。  
5. 最后补一句：方案能提高数据安全性，但也会增加管理与维护复杂度，需要结合业务场景权衡。

#### 模板2：备份恢复方案题
1. 先写业务对数据丢失和停机时间的容忍要求。  
2. 再明确 RPO / RTO 含义，并据此反推备份频率与恢复要求。  
3. 接着给出备份策略：全量、增量 / 差异、日志、异地保存等。  
4. 再写恢复演练、加密保存和验证机制为什么必要。

1. 先识别当前风险最高的告警或业务影响点。
