### 2.3 今日核心知识框架

#### 主线 A：范式与分解

作答顺序固定为：

1. 写出给定函数依赖集；
2. 求候选键；
3. 判断非主属性、主属性；
4. 判断是否存在部分依赖；
5. 判断是否存在传递依赖；
6. 判断最高范式；
7. 若需分解，按依赖或异常原因给出分解结果。

#### 主线 B：SQL 综合

SQL 题先判断类型：

1. 建表 / 约束：`PRIMARY KEY`、`FOREIGN KEY`、`NOT NULL`、`CHECK`；
2. 连接查询：先确定表，再写连接条件；
3. 分组统计：`GROUP BY` 后才能用聚合条件 `HAVING`；
4. 反查：优先考虑 `NOT EXISTS` 或 `LEFT JOIN ... IS NULL`；
5. 插入查询：`INSERT INTO ... SELECT ...`；
6. 更新 / 删除：先确定条件，避免无条件误改。

#### 主线 C：并发控制

并发题固定流程：

1. 找共享数据项；
2. 找冲突操作：不同事务、同一数据项、至少一个写；
3. 画冲突边；
4. 判断有环 / 无环；
5. 若要求加锁，写满足 2PL 的执行序列；
6. 若问死锁，写循环等待与统一锁申请顺序。

#### 主线 D：日志恢复

恢复题固定流程：

1. 找检查点；
2. 列事务状态表：检查点前后、是否写过、是否提交；
3. 已提交且可能未落盘：`Redo`；
4. 未提交且写过数据：`Undo`；
5. 未写数据或检查点前稳定完成且无活动：通常不处理；
6. 恢复数据值时，`Redo` 用新值，`Undo` 回旧值。

### 2.4 必备结论

- 冲突操作三条件：不同事务、同一数据项、至少一个写。
- 冲突图有环，则不是冲突可串行化。
- 2PL 能保证可串行化，但不能保证无死锁。
- 满足 2PL 的序列中，一旦释放锁，就不能再申请新锁。
- 防死锁常用统一锁申请顺序。
- `Read Committed` 能避免脏读，但仍可能不可重复读 / 幻读。
- 范式判断必须先求候选键，不能直接凭感觉判断。
- SQL 分组后筛选聚合结果用 `HAVING`，不是 `WHERE`。
- 已提交写事务通常进入 `Redo`，未提交写事务进入 `Undo`。
- 未提交但未写数据的事务通常不需要撤销数据项更新。

### 2.5 高频考点拆解

#### 2.5.1 范式题：候选键先行

**核心概念：** 范式判断依赖候选键和函数依赖。  
**考法：** 给出关系模式和函数依赖，要求判断最高范式或分解。  
**高频误区：** 不求候选键，直接写 2NF / 3NF。  
**快速判断提示：** 候选键决定“主属性 / 非主属性”，后续判断都依赖它。

#### 2.5.2 SQL 题：别把 WHERE 和 HAVING 混掉

**核心概念：** `WHERE` 过滤分组前行，`HAVING` 过滤分组后的聚合结果。  
**考法：** 查询数量、平均值、最大值、满足统计条件的分组。  
**高频误区：** 在 `WHERE` 中直接写 `COUNT(*) > 1`。  
**快速判断提示：** 出现 `COUNT / SUM / AVG` 的条件，优先看是否应放到 `HAVING`。

#### 2.5.3 反查题：NOT EXISTS 与 LEFT JOIN

**核心概念：** 查询“不存在某类记录”时，常用 `NOT EXISTS` 或左连接空值判断。  
**考法：** 查询没有选课的学生、没有订单的客户、未领取红包的用户。  
**高频误区：** 用普通内连接导致只保留存在匹配记录的行。  
**快速判断提示：** 题干出现“没有 / 未 / 不存在”，先想反查模板。

#### 2.5.4 2PL 执行序列：阶段边界比术语堆砌重要

**核心概念：** 2PL 要求先加锁，后解锁；一旦解锁，不再申请新锁。  
**考法：** 写满足 2PL 的调度，或说明如何降低死锁风险。  
**高频误区：** 写出 `Unlock(A)` 后又 `Lock(B)`。  
**快速判断提示：** 能串行写就先串行写，不要为了“并发感”乱交叉。

#### 2.5.5 恢复题：是否写过数据是关键列

**核心概念：** `Undo` 重点撤销未提交事务已经写入的数据项更新。  
**考法：** 给出事务状态表，判断 Redo / Undo / 不处理。  
**高频误区：** 把“未提交但未写数据”的事务直接列入重点 Undo。  
**快速判断提示：** 恢复表必须有三列：是否提交、是否写过、检查点后是否活跃。

#### 模板2：SQL 分组统计题

```sql
SELECT 分组列, 聚合函数
FROM 表1
JOIN 表2 ON 连接条件
WHERE 分组前过滤条件
GROUP BY 分组列
HAVING 聚合条件;
```

#### 模板3：反查 SQL 题

```sql
SELECT A.*
FROM A
WHERE NOT EXISTS (
  SELECT 1
  FROM B
  WHERE B.外键 = A.主键
);
```

或：

```sql
SELECT A.*
FROM A
LEFT JOIN B ON B.外键 = A.主键
WHERE B.外键 IS NULL;
```

#### 模板4：2PL 执行序列

```text
T1: Lock(A) → Lock(B) → R/W(A) → R/W(B) → Commit → Unlock(B) → Unlock(A)
T2: 等待 T1 释放冲突锁后 → Lock(A) → Lock(B) → R/W(A) → R/W(B) → Commit → Unlock(B) → Unlock(A)
```

### 2.9 易混点速记卡片

- **WHERE：** 分组前过滤行  
  **HAVING：** 分组后过滤聚合结果

- **NOT EXISTS：** 查不存在  
  **INNER JOIN：** 查存在匹配

- **2NF：** 消除部分依赖  
  **3NF：** 消除传递依赖

- **冲突操作：** 不同事务 + 同一数据项 + 至少一个写  
  **非冲突：** 不同数据项不冲突

- **2PL：** 保证可串行化  
  **不保证：** 一定无死锁

- **Redo：** 已提交，怕没落盘  
  **Undo：** 未提交，怕写进去了

**参考答案：**
1. `T1` 已提交，应 `Redo`；`T2` 未提交，应 `Undo`；
2. `X`：保留 / 重做 `T1` 的新值 `80`；
3. `Y`：撤销 `T2` 的更新，恢复为 `50`；
4. `Z`：撤销 `T2` 的更新，恢复为 `30`。

**采分点：**
- T1 Redo；
- T2 Undo；
- X 用新值；
- Y / Z 回旧值。

### 第30题
**参考答案：**
- 范式判断题：先求候选键，再分主属性 / 非主属性，判断部分依赖和传递依赖，最后写最高范式与分解方案。
- SQL 分组统计题：`SELECT 分组列, 聚合函数 FROM ... WHERE ... GROUP BY 分组列 HAVING 聚合条件`。
- 2PL 执行序列题：先按统一顺序申请所需锁，完成读写，提交后释放锁；后一事务等待前一事务释放冲突锁后再进入。
- 日志恢复题：列检查点后活跃情况、是否写过数据、是否提交；已提交写事务 Redo，未提交写事务 Undo，未写或检查点前稳定完成通常不处理。

**采分点：**
- 四类模板都写出核心顺序；
