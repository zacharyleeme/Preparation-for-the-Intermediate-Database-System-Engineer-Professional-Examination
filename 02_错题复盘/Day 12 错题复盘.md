## 二、错题明细

### A. 客观题错题

### 第7题

**题目：**  
设 `R(A, B, C)`，函数依赖集 `F={A → B}`，将 `R` 分解为 `R1(A, B)` 和 `R2(B, C)`，则该分解最可能（ ）

**你的作答：** A  
**标准答案：** B

**错误原因：**  
你把“有公共属性 B”直接近似当成了“可以无损连接”。但本题给定的依赖只有 `A → B`，并不能推出 `B → A` 或 `B → C`，因此公共属性 `B` 不能决定 `R1` 或 `R2` 的全部属性，不满足二元分解常用的无损连接判定条件。

**正确理解应为：**  
- 二元分解先看公共属性；
- 但**有公共属性 ≠ 一定无损**；
- 必须进一步判断公共属性能否决定某个子关系模式。

**纠偏记忆：**  
- 先看 `R1 ∩ R2` 是谁；
- 再看它能不能“定住一边”；
- **公共属性定不住任何一边，就不能轻判无损。**

**建议复习动作：**  
1. 重新口头复述一次：二元分解无损连接的判定口径；  
2. 把第 6 题、第 7 题放在一起对比，专门区分“公共属性是 A”和“公共属性是 B”为什么结果不同；  
3. 再手写 2 道“有公共属性但仍有损”的小题做对照。

### 第15题

**题目：**  
设 `R(A, B, C)`，函数依赖集 `F={A → B}`，将 `R` 分解为 `R1(A, C)` 和 `R2(B, C)`，则该分解最可能（ ）

**你的作答：** B  
**标准答案：** C

**错误原因：**  
你把本题判断成“无损但不保持依赖”，但实际上这题是**既有损，又不保持依赖**：
- 公共属性只有 `C`，而题中并没有 `C → A` 或 `C → B`，因此不能据此判无损；
- 原依赖 `A → B` 也没有在任一子模式中被完整保留，因此也不保持函数依赖。

**正确理解应为：**  
- `R1 ∩ R2 = {C}`；
- `C` 不能决定 `R1(A, C)` 或 `R2(B, C)` 的全部属性；
- 原依赖 `A → B` 被拆散，分解后无法直接在某个子模式中检查。

**纠偏记忆：**  
- **公共属性不够强 → 无损判不出来；**
- **原依赖被拆散 → 保持依赖也要警惕失败。**

**建议复习动作：**  
1. 重新对照第 14 题与第 15 题，分别写出“为什么第 14 题双判正确、第 15 题双判错误”；  
2. 再做 1 组“无损 / 保持依赖”四象限判断小题；  
3. 练习把“公共属性判断”和“原依赖逐条对照”拆成两个独立步骤，不要一次性拍结论。

### B. 下午题 / 主观题正式错题

### 第20题（下午题）

**题型归属：** 无损连接判断

**题目：**  
若将上题关系模式分解为：
- `R1(学号, 课程号, 成绩)`
- `R2(课程号, 教师号)`
- `R3(教师号, 教师姓名)`

请判断该分解是否可以视为合理的无损分解，并简要说明理由。

**你的实际作答：**  
是无损分解，因为R1 ∩ R2 ∩ R3 → R1 或 R1 ∩ R2 ∩ R3 → R2 或 R1 ∩ R2 ∩ R3 → R3 都可以成立

**参考答案：**  
该分解可以视为合理的无损分解。因为：
1. `R1` 与 `R2` 的公共属性为 `课程号`，且 `课程号 → 教师号`，能支持课程与教师信息之间的正确连接；
2. `R2` 与 `R3` 的公共属性为 `教师号`，且 `教师号 → 教师姓名`，能保证教师信息无损连接；
3. 该分解沿原有函数依赖链展开，连接时不会无依据地产生伪组合。

**本题采分点：**
1. 给出“可视为合理无损分解”的结论
2. 识别关键公共属性 `课程号`、`教师号`
3. 能把公共属性与依赖链联系起来说明

**本次已命中的采分点：**
- 给出了“是无损分解”的结论

**本次漏掉的采分点：**
- 没有识别并写出 `课程号`、`教师号` 这两个关键公共属性
- 没有按依赖链说明为什么连接不会失真

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [x] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 本题不是用“三表总交集”去判；
- 应按连接链路分段看公共属性：`R1` 和 `R2` 看 `课程号`，`R2` 和 `R3` 看 `教师号`；
- 该分解沿依赖链自然展开，因而可视为合理的无损分解。

**标准作答路径：**
1. 先写结论：该分解可视为合理的无损分解；
2. 再写 `R1` 与 `R2` 的公共属性为 `课程号`，`R2` 与 `R3` 的公共属性为 `教师号`；
3. 再结合 `课程号 → 教师号`、`教师号 → 教师姓名` 说明连接依据；
4. 最后写“沿原依赖链展开，不会无依据产生伪组合”。

**纠偏记忆：**
- **三表分解题不要上来写 `R1 ∩ R2 ∩ R3`；**
- 先看连接链路，再看每一段公共属性和依赖依据。

**建议复习动作：**
1. 把第 20 题按参考答案重写一遍，限定 4 句话内写完；  
2. 额外练 2 道“三表分解但按链路逐段说明无损”的小题；  
3. 专门把“二元分解判定”与“多表合理无损说明”分开记忆，避免混用公式。

### 第27题（下午题）

**题型归属：** 函数依赖 / 闭包 / 候选键判断

**题目：**  
设关系模式 `R(A, B, C, D)`，函数依赖集为：
- `A → B`
- `B → C`
- `C → D`

请写出该关系模式的候选键，并简要说明依据。

**你的实际作答：**  
A，B，C，D

**参考答案：**  
候选键为 **A**。因为：
- `A → B`
- `B → C`
- `C → D`
所以 `A⁺ = {A, B, C, D}`，A 能决定全部属性；同时 A 为单属性，满足最小性，因此 A 是候选键。

**本题采分点：**
1. 写出候选键 A
2. 写出或说明闭包 `A⁺` 包含全部属性
3. 点明最小性

**本次已命中的采分点：**
- 无

**本次漏掉的采分点：**
- 候选键应写为 `A`，而不是把全部属性并列写出
- 没有写出 `A⁺` 的推出链路
- 没有说明最小性

**失分原因类型：**
- [x] 不会知识点
- [ ] 会概念但不会下手
- [ ] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 候选键不是“把所有属性都写出来”；
- 要找的是**最小**能推出全部属性的属性组；
- 本题中 `A` 顺着依赖链就能推出 `B、C、D`，因此 `A` 本身就是候选键。

**标准作答路径：**
1. 先写候选键为 `A`；
2. 再写 `A⁺ = {A, B, C, D}`；
3. 最后写 A 为单属性，满足最小性。

**纠偏记忆：**
- **候选键 = 最小超键，不是“能想到的属性全写上去”。**

**建议复习动作：**
1. 重新做 5 道“根据依赖链求候选键”的闭包小题；  
2. 每道题都强制写一遍“某属性组闭包 + 是否最小”；  
3. 对比“超键”和“候选键”的区别，避免把“能推出全部属性”和“最小性”拆开记。

### 第29题（下午题）

**题型归属：** 无损连接判断 / 综合分析

**题目：**  
继续以上分解，请判断该分解是否可以视为无损分解，并简要说明你的分析思路。

**你的实际作答：**  
是无损分解，因为 `R1 ∩ R2 ∩ R3 → R1` 或 `R1 ∩ R2 ∩ R3 → R2` 或 `R1 ∩ R2 ∩ R3 → R3`

**参考答案：**  
该分解可以视为合理的无损分解。分析思路可写为：
1. `R1` 与 `R2` 的公共属性为 `B`，且 `B → C`，因此能保证这一步连接不失真；
2. `R2` 与 `R3` 的公共属性为 `C`，且 `C → D`，因此能保证后续连接关系合理；
3. 该分解顺着原依赖链逐段展开，通常可视为无损且结构清晰的分解方案。

**本题采分点：**
1. 给出“可视为合理无损分解”的结论
2. 写出公共属性 B、C
3. 能结合依赖链说明思路

**本次已命中的采分点：**
- 给出了“是无损分解”的结论

- 没有写出两段公共属性 `B`、`C`
- 没有结合 `B → C`、`C → D` 解释连接链路

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [x] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 本题仍应按分段连接思路写，而不是写三表总交集；
- 先看 `R1` 与 `R2` 的公共属性 `B`，再看 `R2` 与 `R3` 的公共属性 `C`；
- 最后再把依赖链和“合理无损”结论合起来写完整。

**标准作答路径：**
1. 先写结论：该分解可以视为合理的无损分解；
2. 写 `R1 ∩ R2 = {B}`，并结合 `B → C` 说明第一段连接；
3. 写 `R2 ∩ R3 = {C}`，并结合 `C → D` 说明第二段连接；
4. 最后写“分解顺着依赖链展开，结构清晰，连接合理”。

**纠偏记忆：**
- **链式分解题，按链路一段一段写；不要偷换成三表总交集判定。**

### 第30题（下午题）

**题型归属：** 标准答题模板复写

**题目：**  
请用“先写原依赖 → 再写分解结果 → 再判保持依赖 / 无损连接 → 最后给结论”的顺序，写出一段 4～6 句的简洁标准答案。

**你的实际作答：**  
- 原关系模式存在传递函数依赖的问题
- 将原函数分解为 R1、R2、R3 ，消除了传递函数依赖
- 并且因为分解后的关系模式符合 `R1 ∩ R2 ∩ R3 → R1` 或 `R1 ∩ R2 ∩ R3 → R2` 或 `R1 ∩ R2 ∩ R3 → R3` 所以是无损分解
- 该分解方式降低了属性冗余，并且消除了传递函数依赖

**参考答案：**  
原关系模式的主要函数依赖为 `A → B`、`B → C`、`C → D`。将其分解为 `R1(A, B)`、`R2(B, C)`、`R3(C, D)` 后，这三条依赖分别保留在对应子关系模式中，因此该分解保持函数依赖。进一步看，分解沿原依赖链逐步展开，公共属性可以支撑连接过程，因此可视为合理的无损分解。综上，该分解既较好保留了原依赖，也使结构更清晰。

**本题采分点：**
1. 顺序完整：原依赖 → 分解结果 → 保持依赖 → 无损判断 → 结论
2. 句子结构清楚
3. 至少包含两个判断结论和对应理由

**本次已命中的采分点：**
- 基本写出了“分解结果 + 无损结论 + 最终评价”的部分内容

**本次漏掉的采分点：**
- 没有先写出原依赖 `A → B`、`B → C`、`C → D`
- 没有明确写出“保持函数依赖”的判断及理由
- 无损判断仍误用了三表总交集的写法，理由不规范

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [x] 只写结论没写依据

- 顺序必须稳定：原依赖 → 分解结果 → 保持依赖 → 无损连接 → 总结；
- 只要顺序乱、关键结论漏掉、无损口径写错，就会明显丢分。

**标准作答路径：**
1. 先列原依赖：`A → B`、`B → C`、`C → D`；
2. 再写分解结果：`R1(A, B)`、`R2(B, C)`、`R3(C, D)`；
3. 写保持依赖：三条依赖分别保留在对应子模式中；
4. 写无损连接：分解沿依赖链展开，公共属性能支撑连接；
5. 最后总结：该分解既保持依赖，又可视为合理无损，结构更清晰。

**纠偏记忆：**
- **模板题不是“想到什么写什么”，而是必须按固定句序落笔。**

**建议复习动作：**
1. 把第 30 题按参考模板重写 2 遍，一遍 6 句版，一遍 4 句版；  
2. 口头背熟“原依赖 → 分解结果 → 保持依赖 → 无损连接 → 结论”这条句序；

## 三、本次练习暴露出的关键易错点

### 1. 知识判断层面的易错点

- 二元分解里，仍会把“有公共属性”误近似成“无损连接”；
- 候选键判断偶尔会从“最小超键”滑成“把相关属性全写上去”；
- 会记定义，但一到具体依赖链就容易漏掉“闭包”和“最小性”两个硬步骤。

### 2. 下午题书写层面的易错点

- 多表分解题中，容易把分析过程偷换成 `R1 ∩ R2 ∩ R3` 这种不规范写法；
- 容易直接写“无损”“保持依赖”结论，却不写公共属性、依赖链和分段说明；
- 模板题还没有真正形成稳定句序，写起来仍然偏“想到哪写到哪”。

### 3. 采分点意识层面的易错点

- 候选键题已经能抓到核心结论，但常漏写“闭包推出路径 + 最小性”；
- 综合评价题知道大方向，却没把“减少冗余 / 异常 / 可恢复业务信息”写成标准术语；
- 下午题当前最大问题不是完全不会，而是**会一点、写不满、采分点掉得快**。

## 四、Day 12 需要继续巩固的重点

1. 二元分解无损连接的固定判断顺序：先找公共属性，再看能否决定一边
2. 多表链式分解的说明口径：按连接链路逐段写，不写三表总交集
3. 候选键判断时必须同时写“闭包 + 最小性”
4. 保持函数依赖题必须先列原依赖，再逐条对照子模式
5. 标准答题模板要写成固定句序：原依赖 → 分解结果 → 保持依赖 → 无损连接 → 结论
6. 综合评价题要主动补上“减少冗余、降低更新异常、连接可恢复业务信息”等规范术语
7. 对“无损连接”和“保持函数依赖”继续做并列对照，避免概念混写
