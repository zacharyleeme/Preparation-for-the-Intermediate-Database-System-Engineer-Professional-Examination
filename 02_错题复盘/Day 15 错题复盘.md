### B. 下午题 / 主观题正式错题

### 第27题（下午题）

**题型归属：** 检查点与事务集合判断

**题目：**  
某系统在日志中记录了一个检查点，检查点时活跃事务为 `T4、T5`。之后：

- `T4` 提交
- `T5` 未提交
- `T6` 启动并提交
- 此时系统崩溃

请说明在恢复分析时，为什么至少要重点关注 `T4、T5、T6` 的状态。

**你的实际作答：**  
- 因为检查点创建时活跃事务为T4、T5。在有检查点的情况下恢复时应从检查点附近开始日志分析。
- 
- 

**参考答案：**  
因为检查点时系统中活跃事务为 `T4、T5`，说明它们跨越了检查点，恢复时需要重点确认其后续是否提交。检查点之后 `T6` 又启动并执行，因此它也属于故障前需要纳入恢复分析的事务集合。换言之，恢复时至少要关注“检查点时仍活跃”以及“检查点后新出现”的事务状态。

**本题采分点：**
1. 写出 `T4、T5` 在检查点时处于活跃状态
2. 写出 `T6` 是检查点后新出现的事务
3. 能说明这些事务都需要纳入恢复分析范围

**本次已命中的采分点：**
- 写到了 `T4、T5` 在检查点时处于活跃状态

**本次漏掉的采分点：**
- 没有写出 `T6` 是检查点后新出现、同样需要重点核查状态的事务
- 没有把“检查点时仍活跃 + 检查点后新出现”的事务都应纳入恢复分析范围这层逻辑写完整
- 答案停在“从检查点附近开始分析”，但没有回到题目真正追问的“为什么要重点关注 `T4、T5、T6`”

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 检查点题不只是回答“从哪里开始扫日志”；
- 还要回答“哪些事务必须被纳入恢复判断集合”；
- 本题里，`T4、T5` 是检查点时仍活跃的事务，`T6` 是检查点后新出现的事务，这三者都与故障前状态直接相关，因此至少都要重点核查其提交 / 未提交状态。

**标准作答路径：**
1. 先写：`T4、T5` 在检查点时处于活跃状态，恢复时必须继续确认它们后续是否提交；
2. 再写：`T6` 虽然不在检查点时活跃事务列表里，但它在检查点后启动并在故障前执行，因此同样属于恢复分析对象；
3. 最后写：恢复时至少要覆盖“检查点时仍活跃”与“检查点后新出现”的事务集合，才能正确判断哪些事务应 Undo、哪些事务应 Redo。

**纠偏记忆：**
- **检查点题先分两类事务：检查点时仍活跃的、检查点后新出现的；两类都要纳入恢复分析。**

**建议复习动作：**
1. 把本题按参考答案重写 2 遍；  
2. 再补 2 道“检查点 + 活跃事务集合 + 恢复范围”小题；  
3. 下次遇到检查点题，先写“哪些事务要看”，再写“从哪里开始看”。

## 三、本次练习暴露出的关键易错点

### 1. 知识判断层面的易错点

- 日志恢复基础判断已经明显比前几天更稳，但一遇到“检查点 + 活跃事务集合”这类题，仍可能只记得“从检查点附近开始分析”，却漏掉“哪些事务必须纳入恢复范围”；
- 对介质故障、系统故障、立即修改、延迟修改这些概念的大方向已经能分清，但个别术语表述还不够精确；
- 对“为什么需要 Undo / Redo”的判断已经基本建立，但要继续把“事务集合判定”与“恢复动作判定”连成完整一条线。

### 2. 下午题书写层面的易错点

- 方向对了之后，仍有“写到一半就停”的问题，尤其容易漏掉最后一句总括结论；
- 检查点类题目容易把答案写成“恢复起点题”，而没有完整回答“恢复对象题”；
- 比较题和模板题已经能写出主要内容，但还需要进一步把“关键差异句”和“标准收束句”写稳。

### 3. 采分点意识层面的易错点

- 当前最大问题已经不是“完全不会恢复题”，而是**大方向会了，但个别题的采分点覆盖还不够满**；
- 诸如“数据库文件本身不可用”“立即修改”“检查点后新出现事务”“恢复到只保留已提交事务结果的正确状态”这类短语，今后要尽量直接写出来；
- 综合题如果已经写出风险、手段、原因，就要逼自己再补最后一句结论，避免 2/3 式失分反复出现。

## 四、Day 15 需要继续巩固的重点

1. 检查点题的固定顺序：**先看哪些事务要纳入恢复分析，再看从哪里开始分析日志**
2. 活跃事务集合判断：分清“检查点时仍活跃的事务”和“检查点后新出现的事务”
3. 介质故障题的标准口径：**数据库文件本身不可用 → 先恢复备份 → 再结合日志重做已提交事务**
4. 立即修改与延迟修改的标准术语，避免把“立即修改”误写成“立即提交”
5. 恢复题的标准收束句：**数据库应恢复到只保留已提交事务结果的正确状态**
6. 综合表达题的固定句序：**当前风险 → 日志 / 检查点 / WAL 手段 → 为什么有效 → 结论**
7. 比较题的固定写法：两边都写、差异点单独写、最后补“关键差异在于提交前数据是否可能已写库”
