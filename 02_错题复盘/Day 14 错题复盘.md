## 二、错题明细

### A. 客观题正式错题

### 第3题（客观题）

**题型归属：** 并发基础概念

**题目：**  
多个事务在时间上交替执行、提高资源利用率的处理方式称为（A ）

**你的实际作答：**  
A. 串行执行

**正确答案：**  
B. 并发执行

**错误原因：**  
把“多个事务在时间上交替执行、提高资源利用率”的描述误判成了串行执行，说明对“串行”和“并发”的基本定义还存在混淆。

**正确理解应为：**  
- **串行执行**：一个事务全部完成后，再执行下一个事务；
- **并发执行**：多个事务在时间上交替推进，以提高吞吐量和资源利用率。

**纠偏记忆：**  
- **交替推进是并发；一个做完再下一个才是串行。**

**建议复习动作：**  
1. 把“串行执行 / 并发执行”各写 1 句定义；  
2. 再口头复述 3 次：“并发不等于同时完成，而是时间上交替执行”；  
3. 补做 2 道“并发 vs 串行”基础辨析题。

### 第5题（客观题）

**题型归属：** 并发异常识别

**题目：**  
事务 T1 读取了事务 T2 尚未提交的数据，随后 T2 回滚。T1 遇到的现象称为（A ）

**你的实际作答：**  
A. 丢失修改

**正确答案：**  
B. 脏读

**错误原因：**  
把“读到了未提交数据”误写成“丢失修改”，说明对并发异常的判定入口还不够稳。

**正确理解应为：**  
- **脏读**：读到了其他事务**尚未提交**的数据；
- **丢失修改**：两个事务都修改同一数据项，后一次更新把前一次结果覆盖掉。

**纠偏记忆：**  
- **看到‘未提交还被读到’，先想脏读；看到‘一个修改把另一个改没了’，再想丢失修改。**

**建议复习动作：**  
1. 把“脏读 / 不可重复读 / 幻读 / 丢失修改”各写 1 句最短定义；  
2. 再把它们分别配 1 个业务场景；  
3. 明天学习前先口头区分一次这 4 个异常。

### 第12题（客观题）

**题型归属：** 两段锁协议（2PL）

**题目：**  
关于两段锁协议（2PL），下列说法正确的是（A ）

**你的实际作答：**  
A. 事务可以先解锁再继续申请新锁

**正确答案：**  
C. 事务进入解锁阶段后不能再申请新锁

**错误原因：**  
把两段锁协议的核心限制写反了。2PL 最关键的硬规则，正是“一旦开始解锁，就不能再申请新锁”。

**正确理解应为：**  
- **扩展阶段**：只能加锁，不能解锁；
- **收缩阶段**：只能解锁，不能再加锁。

**纠偏记忆：**  
- **2PL 不是“先解再加”，而是“开始解了，就别再加”。**

**建议复习动作：**  
1. 把“扩展阶段 / 收缩阶段”各写 1 句定义；  
2. 再对照 Day 14 第 27 题，专门练 2 道“是否违反 2PL”的判断题；  
3. 进入 Day 15 前，先默写一次：“解锁后不能再加锁”。

### B. 下午题 / 主观题正式错题

### 第27题（下午题）

**题型归属：** 两段锁协议判断

**题目：**  
事务 T1 的加锁 / 解锁顺序如下：

- `S(A)`
- `X(B)`
- `UNLOCK(A)`
- `S(C)`
- `UNLOCK(B)`
- `UNLOCK(C)`

请判断该事务是否符合两段锁协议，并说明原因。

**你的实际作答：**  
- 符合两段锁协议
- S(A)为共享锁，添加共享锁后，只能查询数据A，不能修改该数据
- X(B)为排它锁，添加排它锁后，不能查询、修改该数据
- S(C)为共享锁，添加共享锁后，只能查询数据C，不能修改该数据
- 在事务T1的加解锁过程中，没有产生冲突

**参考答案：**  
该事务 **不符合两段锁协议**。因为它在 `UNLOCK(A)` 之后又申请了新锁 `S(C)`。按照两段锁协议，事务一旦开始解锁，就进入收缩阶段，之后不能再申请新的锁。

**本题采分点：**
1. 写出“不符合 2PL”
2. 指出 `UNLOCK(A)` 后又申请 `S(C)`
3. 说明违反的是“解锁后不能再加锁”

**本次已命中的采分点：**
- 本题未命中正式采分点

**本次漏掉的采分点：**
- 把结论误判为“符合两段锁协议”
- 没有抓住 `UNLOCK(A)` 之后又申请 `S(C)` 这一关键违规点
- 把注意力放在 S 锁 / X 锁的含义上，但没有回到 2PL 的阶段规则本身

**失分原因类型：**
- [x] 不会知识点
- [ ] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 判断 2PL 时，先不要展开讲 S 锁 / X 锁的通用定义；
- 先盯住“有没有出现**解锁后再加锁**”；
- 本题在 `UNLOCK(A)` 之后出现 `S(C)`，已经直接违反两段锁协议。

**标准作答路径：**
1. 先写结论：该事务不符合两段锁协议；
2. 再指出：事务在 `UNLOCK(A)` 之后又申请了新锁 `S(C)`；
3. 最后补一句：按照 2PL，进入解锁阶段后不能再申请新锁，因此违反协议。

**纠偏记忆：**
- **判断 2PL，先找“解锁后又加锁”；找到了，直接判不符合。**

**建议复习动作：**
1. 把本题按参考答案重写 2 遍；  
2. 再补 3 道“给加锁 / 解锁序列，判断是否符合 2PL”的小题；  
3. 把“锁类型判断”和“2PL 阶段判断”拆开练，避免混成一题。

## 三、本次练习暴露出的关键易错点

### 1. 知识判断层面的易错点

- “串行执行”和“并发执行”的基本定义还不够稳，看到“交替执行、提高资源利用率”时仍可能误判；
- 并发异常的首要识别词还需要继续固化，尤其是：
  - **未提交被读到 → 脏读**
  - **同一行前后不一样 → 不可重复读**
  - **同一条件结果集变了 → 幻读**
- 两段锁协议的硬规则还没有完全内化，仍可能把“解锁后又加锁”的典型违规序列看成正确。

### 2. 下午题书写层面的易错点

- 有些题已经能判断方向，但标准答案句序还不够稳定，容易出现“知道怎么做，但没有按采分点顺序写”；
- 改进方案题有时会把多个异常一起写进去，导致答案焦点漂移；
- 综合表达题容易写出“局部风险 + 局部手段”，但没有把总结句收住，导致答案显得不完整。

### 3. 采分点意识层面的易错点

- 当前最典型的问题已经不是“完全不会”，而是**基础判断能做、但标准表达和采分点覆盖还不稳定**；
- 对“优先图关系”“等价串行顺序”“为什么该手段有效”这类收尾句，还需要形成每题必写的习惯动作；
- 对“幻读”的控制方向，不能只停留在一般共享锁层面，要更敏感地联想到**结果集变化、范围控制、隔离级别提升**。

## 四、Day 14 需要继续巩固的重点

1. 串行执行与并发执行的定义差异，确保看到题干关键词能立即落到正确概念
2. 脏读、不可重复读、幻读、丢失修改四类异常的最短判定口径
3. 冲突可串行化题的固定顺序：**先找冲突操作 → 再写边 → 再看有无环 → 最后写结论**
4. 两段锁协议的固定口径：**一旦开始解锁，就不能再申请新锁**
5. 锁类型判断与 2PL 判断要分开做，不要把“锁的含义”误当成“协议是否满足”的判断依据
6. 幻读题的控制方向要优先想到更高隔离级别、必要时串行化或更严格范围控制
7. 综合表达题必须写完整：**风险 → 手段 → 原因 → 结论**，不要只写前 2～3 句就停
