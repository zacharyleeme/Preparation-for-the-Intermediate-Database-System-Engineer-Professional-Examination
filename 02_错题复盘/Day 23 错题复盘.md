## 二、错题明细

### A. 客观题正式错题

#### 第9题

- **你的实际作答：** A  
- **正确答案：** B  
- **错因：** 把“物业经理姓名”的决定侧误判成了“人员编号”。在“人员分配（人员编号，人员姓名，小区编号，物业经理姓名，人员职责）”这类题里，**物业经理姓名更稳的决定侧是小区编号**，而不是人员编号。  
- **正确理解：** 这类题做单属性依赖判断时，要先看“这个属性更像挂在哪个实体身上”。“人员姓名”挂在人员侧，“物业经理姓名”挂在小区侧。  
- **建议复习动作：** 把“人员编号 → 人员姓名”“小区编号 → 物业经理姓名”手写 3 遍，再回看第 23～26 题。

#### 第10题

- **你的实际作答：** B  
- **正确答案：** C  
- **错因：** 题干给了 `车号→组别`、`手机号→车手姓名`，这类信息在冲线记录表里首先暴露的是**传递依赖与冗余风险**，而不是“仅参照完整性问题”。  
- **正确理解：** 只要题干已经明确某个属性可以稳定决定另一个属性，就要先警惕“冗余存储、更新异常、3NF 不稳”，不能优先往实体完整性 / 参照完整性上跳。

### C. 下午题 / 主观题正式错题

#### 第20题（下午题）

**题型归属：** 候选键与范式判断  
**你的实际作答：** 把“身份证号、医保卡号”写成了组合候选键，并据此判断存在部分依赖，结论写成“最高只符合第一范式”。  
**参考答案：** 候选键应为 **身份证号、医保卡号** 两个**单独的候选键**；其他属性完全依赖于它们，不存在组合键下的部分依赖，因此至少可稳定达到 **3NF**，按更严格口径通常也可视为满足 **BCNF**。

**本题采分点：**
1. 写出全部候选键；
2. 说明为什么不存在部分依赖；
3. 给出正确的范式结论。

**本次已命中的采分点：**
- 写出了“身份证号、医保卡号”这两个候选键名称。

**本次漏掉的采分点：**
- 把两个单属性候选键误写成一个组合候选键；
- 因误判候选键，进一步错误写出了“部分依赖”；
- 范式结论整体被带偏。

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- “身份证号唯一”“医保卡号唯一”表示它们各自都可以单独唯一标识患者；
- **多个单属性候选键 ≠ 一个组合候选键**；
- 既然不存在组合候选键场景，就不能直接往“部分依赖 / 不满足 2NF”上写。

**标准作答路径：**
1. 先列出：候选键为身份证号、医保卡号；
2. 再写：其他属性完全依赖于这两个候选键；
3. 再判断：不存在组合键下的部分依赖，也无题干给出的不合规传递依赖；
4. 最后下结论：至少达到 3NF，按更严格口径可视为 BCNF。

**纠偏记忆：**
- “两个能单独唯一标识的字段，是两个候选键，不是一个拼起来的候选键。”

**建议复习动作：**
1. 把“单属性候选键 / 组合候选键 / 超键”的区别口头复述 1 遍；
2. 重写第 20 题完整答案 1 遍；
3. 再做一遍“先求候选键，再判 2NF / 3NF”的模板题。

#### 第23题（下午题）

**题型归属：** 函数依赖识别  
**你的实际作答：** 写了“人员编号→人员姓名”，但同时又写了“人员编号→物业经理姓名”“小区编号→人员编号”“人员编号→人员职责”。  
**参考答案：** 稳定成立的单属性左侧函数依赖应优先写为：**人员编号→人员姓名**、**小区编号→物业经理姓名**。

**本题采分点：**
1. 写出人员编号决定人员姓名；
2. 写出小区编号决定物业经理姓名；
3. 不把“人员职责”误写成只依赖单属性。

**本次已命中的采分点：**
- 写出了“人员编号 → 人员姓名”。

**本次漏掉的采分点：**
- 没有写稳“小区编号 → 物业经理姓名”；
- 把“人员职责”误写成可由人员编号单独决定；
- 写入了不够稳的“小区编号 → 人员编号”。

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- “人员姓名”是人员实体静态信息，稳定挂在人员编号上；
- “物业经理姓名”更稳地挂在小区一侧；
- “人员职责”表示“某个人在某个小区负责什么工作”，本质上是**联系属性**，应依赖 `(人员编号, 小区编号)`，不能直接降成单属性依赖。

**标准作答路径：**
1. 先把静态实体属性和联系属性分开看；
2. 先写最稳的单属性依赖：人员编号→人员姓名，小区编号→物业经理姓名；
3. 对“人员职责”这类跨实体发生的信息，保留到组合键关系中判断。

**纠偏记忆：**
- “挂在实体身上的写单属性依赖，挂在联系上的不要乱降维。”

**建议复习动作：**
1. 把“实体属性 / 联系属性”的判断口头复述 1 遍；
2. 重写第 23 题；
3. 立刻连做第 24、25、26 题，检查后续是否继续被这个误判带偏。

**参考答案：** 更合理的 3NF 分解方向应为：**人员（人员编号，人员姓名）**、**小区（小区编号，物业经理姓名）**、**人员分配（人员编号，小区编号，人员职责）**。

**本题采分点：**
1. 拆出人员信息表；
2. 拆出小区信息表；
3. 保留人员与小区之间的分配关系表。

**本次已命中的采分点：**
- 拆出了人员信息表；
- 拆出了小区信息表。

**本次漏掉的采分点：**
- 没有保留“人员分配”这张真正承载业务关系的表；
- 把“人员职责”错误塞回了人员静态信息表。

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**

**标准作答路径：**

2. 再拆小区静态信息；
3. 最后保留人员分配关系表，把职责放在这张表里。

**纠偏记忆：**

**建议复习动作：**

2. 给每张表补一句“为什么放这些字段”；
3. 再衔接第 26 题写主外键。

#### 第26题（下午题）

- 写出了人员信息表主键是人员编号；
- 写出了小区信息表主键是小区编号。

**本次漏掉的采分点：**
- 漏掉了人员分配表这张核心关系；
- 没有写出 `(人员编号, 小区编号)` 这个复合主键；

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词

**纠偏记忆：**

#### 第29题（下午题）

**题型归属：** 3NF 判断与依赖说明  
**你的实际作答：** 把候选键写成了“(序列号，车号，手机号)”，又把问题写成“部分依赖”，最后得出“不满足 2NF，也不满足 3NF”。  
**参考答案：** 更稳的主码是 **序列号**；关键依赖链应写为：`序列号 → 车号` 且 `车号 → 组别`，`序列号 → 手机号` 且 `手机号 → 车手姓名`。这说明存在通过非主属性继续决定其他属性的情况，属于 3NF 不稳场景。

**本题采分点：**
1. 写出至少 1 组关键依赖链；
2. 写明这是导致 3NF 不满足的原因；
3. 把冗余 / 异常与依赖链联系起来。

**本次已命中的采分点：**
- 写到了“车号可以决定车型和组别”“手机号可以决定车手姓名”这两个方向。

**本次漏掉的采分点：**
- 主码没有收准到“序列号”；
- 把传递依赖误写成了部分依赖；
- 没有把“序列号 → 车号 / 手机号”这一步补完整，导致依赖链不闭合。

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 这题不该先上来拼一个大候选键；
- 题干已明确“序列号唯一确定一条冲线记录”，所以主码先稳到序列号；
- 之后再把 `序列号 → 车号 → 组别`、`序列号 → 手机号 → 车手姓名` 这两条链写出来，才有资格判断 3NF。

**标准作答路径：**
1. 先写主码：序列号；
2. 再写依赖链：序列号 → 车号 → 组别，序列号 → 手机号 → 车手姓名；
3. 再下结论：存在通过非主属性继续决定其他属性的情况，不满足更稳妥的 3NF；
4. 最后补一句：容易造成冗余与更新异常。

**纠偏记忆：**
- “题干说谁唯一，就先把主码钉住谁；不要把能引用到的字段都拼进候选键。”

**建议复习动作：**
1. 重写第 29 题；
2. 把“部分依赖”和“传递依赖”的判断条件各写 2 句对比；
3. 再接着重写第 30 题模板答案。

#### 第30题（下午题）

**题型归属：** 标准模板复写 / 规范化说明题  
**你的实际作答：** 基本按“主码 / 依赖 / 结论 / 分解”顺序写了，但主码仍写成“(序列号，车号，手机号)”，依赖问题仍按“部分依赖”去写，结尾价值句也没有完全收稳。

1. 顺序完整：主码 → 依赖问题 → 范式结论 → 分解方向与价值；

**本次已命中的采分点：**

**本次漏掉的采分点：**

- 结尾缺少“减少冗余、提高一致性、便于维护”等完整价值收束句。

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

## 三、本次练习暴露出的关键易错点

### 1. 知识判断层面的易错点

- **多个单属性候选键 vs 组合候选键** 这一层仍不稳。只要题干给了“身份证号唯一”“医保卡号唯一”这种口径，就要优先想到“各自都是候选键”，而不是把它们强行拼成组合键；
- **部分依赖 vs 传递依赖** 仍容易混。Day 23 最典型的问题是：有些题本来该写“`序列号 → 车号 → 组别` 这类传递依赖链”，结果被写成了“部分依赖”；
- **实体属性 vs 联系属性** 的边界仍不够稳。“人员职责”这种属性，不属于人员静态信息，也不属于小区静态信息，而是属于“人员—小区分配关系”本身。

### 2. 下午题书写层面的易错点

- 现在的主要问题不是“完全不会写”，而是**第一步经常写偏，导致后面整条链一起漂**：
  - 候选键一旦写偏，2NF / 3NF 判断就会跟着偏；
  - 单属性依赖一旦乱写，后面的分解表结构也会跟着错位；
  - 模板题一旦主码没钉住，依赖链和结尾价值句都容易发虚；
- 规范化分解题当前最典型的问题是：**知道要拆，但拆完以后把“联系关系”丢了，或者把联系属性塞回实体表里**；
- 主键 / 外键题当前最需要修的是：**先立实体表主键，再用联系表承载外键关系**，不要让两个实体表互相套外键。

### 3. 采分点意识层面的易错点

- 题干只要出现“**唯一**”，就要优先从候选键角度想，不要先从“拼起来会不会更保险”去想；
- 题干只要出现“**某人 / 某物在某场景下承担什么角色 / 职责 / 数量 / 明细**”，就要优先想到这类属性很可能属于**联系关系**；
- 题干只要出现“**序列号唯一 / 编号唯一**”，就要先把主码钉住，再补依赖链；
- 范式题最后一定要补一句“**为什么拆、拆完有什么价值**”，否则答案容易停留在“会拆表，但不会收束”。
