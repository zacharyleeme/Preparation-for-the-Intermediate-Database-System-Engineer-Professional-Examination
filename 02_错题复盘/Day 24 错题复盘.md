## 二、错题明细

### A. 客观题正式错题

#### 第12题

- **你的实际作答：** A
- **正确答案：** D
- **错因：** 漏掉了候选键 `AC`。你只看到了 `AB → C`，但没有继续利用 `C → B` 反推 `AC → B`，因此把“两个候选键”误缩成了“一个候选键”。
- **正确理解：**
  - `AB⁺ = {A, B, C}`，所以 `AB` 是候选键；
  - 又因为 `C → B`，所以 `AC⁺ = {A, C, B}`，因此 `AC` 也是候选键；
  - 所以本题候选键应为 **`AB` 和 `AC`**。
- **建议复习动作：**
  1. 把 `AB⁺`、`AC⁺` 的闭包各手写 1 遍；
  2. 再对照下午题 **第23题、第24题、第25题**，把这组依赖链连起来复盘；
  3. 强化一句话：**“出现 `X→Y` 后，要继续检查能否补出新的候选键，不要只盯第一眼那个组合键。”**

1. 写出候选键 `(学号, 课程号)`；
2. 写出依赖推出链路；

**本次已命中的采分点：**
- 写出了 `(学号, 课程号)` 这组核心组合。

**本次漏掉的采分点：**
- 把 `教师号` 误写成候选键；
- 没有把 `课程号 → 教师号 → 教师姓名` 这段推出链补完整；

- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范

- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- `教师号` 只能推出 `教师姓名`，不能推出 `学号、课程号、成绩`，所以它不是候选键；

## 三、本次练习暴露出的关键易错点

### 1. 候选键补全意识仍不稳

Day 24 最集中的问题，是你已经能看到“一个大体正确的候选键”，但还不够会做两件事：

1. **继续检查是否存在第二个候选键**（如第12题、第23题漏掉 `AC`）；
2. **克制住往候选键里额外塞字段**（如第19题、第27题）。

一句话纠偏：

> **候选键要么是漏了，要么是写多了；真正稳的写法，是“先求闭包，再讲最小性”。**

### 2. 分解方案最容易漏“中间那张承接依赖的表”

第21题漏掉 `课程号 → 教师号` 对应的表，第29题漏掉 `成员号 → 部门号` 在成员表中的保留，本质上是同一种问题：

> **分解时只顾着把两头拆开，却把中间真正承接依赖链的那一层漏掉了。**

后续做规范化分解题时，建议固定自查：
- 原依赖是否逐条都能找到落点？
- 中间桥梁依赖是否有独立表或被正确保留？
- 一旦漏掉桥梁层，“保持依赖”几乎一定站不住。

### 3. “无损连接”与“保持函数依赖”仍容易混写

第22题和第25题最明显：
- 你已经知道这两者都属于“分解质量判断”；
- 但一到真正写答案时，还是容易把它们揉成一句。

一句话纠偏：
- **无损连接：** 看连回去会不会失真；
- **保持函数依赖：** 看原来的函数依赖有没有丢。

### 4. 联系属性识别在进步，但还没完全带动候选键与分解链同步稳定

好消息是：第27题、第28题已经明显显示你能认出“分工角色”属于联系属性。  
但问题是：
- 候选键还会多写；
- 分解后成员表仍会漏 `部门号`；
- 说明“联系属性归位”这条线虽有进步，但还没有完全反推到整条分解链上。
