### 第20题（下午题）

**题型归属：** 范式判断 / 规范化分析

**题目：**  
继续上题，请判断该关系模式最高达到几范式，并说明原因。

**你的实际作答：**  
- 2NF，因为学号和课程号都不能单独决定成绩，所以符合第二范式
- 教师号依赖于课程号，教师姓名依赖于教师号，所以不符合第三范式

**参考答案：**  
该关系模式最高只能达到 **1NF**。因为候选键为 `(学号, 课程号)`，而非主属性 `教师号` 只依赖于 `课程号`，属于对组合键的部分函数依赖；同时 `教师号 → 教师姓名` 还体现出传递依赖。因此它不能达到 2NF，更不可能达到 3NF / BCNF。

**本题采分点：**
1. 写出最高范式为 1NF
2. 指出 `课程号 → 教师号` 导致部分依赖
3. 能提及 `教师号 → 教师姓名` 的传递依赖问题

**本次已命中的采分点：**
- 提到了 `教师号 → 教师姓名` 这条传递依赖链

**本次漏掉的采分点：**
- 最高范式误判为 2NF，核心结论错误
- 没有把 `课程号 → 教师号` 明确识别为“对组合键的部分函数依赖”

**失分原因类型：**
- [x] 不会知识点
- [ ] 会概念但不会下手
- [ ] 会分析但写不规范
- [x] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 候选键是 `(学号, 课程号)`；
- `课程号 → 教师号` 说明非主属性 `教师号` 只依赖组合键的一部分，因此已经不满足 2NF；
- `教师号 → 教师姓名` 进一步体现出传递依赖，但在判断最高范式时，前面的部分依赖已经足以把它卡在 1NF。

**标准作答路径：**
1. 先写候选键为 `(学号, 课程号)`；
2. 再写 `课程号 → 教师号` 属于部分函数依赖，因此不能达到 2NF；
3. 再补 `教师号 → 教师姓名` 体现传递依赖；
4. 最后给出结论：最高仅为 1NF。

**纠偏记忆：**
- **先卡 2NF，再谈 3NF；前一层过不去，后面不用往上抬。**

**建议复习动作：**
1. 重新做 3 道“组合键 + 部分依赖”判最高范式的小题；  
2. 每题都强制写“候选键 → 部分依赖 / 传递依赖 → 最高范式”；  
3. 对照 Day 13 第 19 题和第 20 题，练会“先求键，再判范式”的固定顺序。

### 第23题（下午题）

**题型归属：** 候选键判断 / 3NF 与 BCNF 区分

**题目：**  
设关系模式 `R(A, B, C)`，函数依赖集为：
- `AB → C`
- `C → B`

请写出该关系模式的候选键，并简要说明依据。

**你的实际作答：**  
- 候选键为 AB ，因为C依赖于 AB ，且A、B都不能单独推导到 C。所以AB才是关系模式的候选键

**参考答案：**  
候选键为 **AB** 和 **AC**。因为：
- `AB → C`，所以 `AB⁺ = {A, B, C}`，故 `AB` 能决定全部属性；
- 由 `C → B` 可得 `AC → B`，因此 `AC⁺ = {A, C, B}`，故 `AC` 也能决定全部属性；
- `A`、`B`、`C` 单独都不能决定全部属性，因此 `AB` 和 `AC` 是候选键。

**本题采分点：**
1. 写出候选键 `AB`、`AC`
2. 能写出闭包或推出路径
3. 说明最小性

**本次已命中的采分点：**
- 写出了 `AB` 这一组候选键方向

**本次漏掉的采分点：**
- 漏掉了第二个候选键 `AC`
- 没有写出 `AC` 通过 `C → B` 推出全部属性的链路
- 最小性说明不完整

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 本题的陷阱不在 `AB`，而在**不要漏掉 `AC`**；
- 因为 `AC` 中已有 `C`，再由 `C → B` 可以补出 `B`，所以 `AC` 也能推出全部属性；
- 候选键题一定要把“所有候选键”枚举完整，而不是写出一个就停。

**标准作答路径：**
1. 先试 `AB` 的闭包，写出 `AB⁺ = {A, B, C}`；
2. 再试 `AC` 的闭包，写出 `AC⁺ = {A, C, B}`；
3. 最后说明 `A`、`B`、`C` 单独都不行，因此候选键为 `AB`、`AC`。

**纠偏记忆：**
- **候选键题要防“只找到一个就停”的漏键错误。**

**建议复习动作：**
1. 额外补 5 道“多候选键”小题，专门训练枚举完整性；  
2. 每题都至少写两组可能键的闭包，不要只验第一组；  
3. 把本题和第 24 题放一起看，理解为什么漏掉 `AC` 会直接影响后续 3NF 判断。

### 第26题（下午题）

**题型归属：** 综合评价 / 规范化思路对比

**题目：**  
请用 2～4 句概括：为什么说 BCNF 更严格，但考试中仍常常先讲 3NF 分解思路。

**你的实际作答：**  
- 因为拆分成BCNF 后可能会破坏原有依赖关系，

**参考答案：**  
BCNF 更严格，因为它要求每个非平凡函数依赖的决定因素都必须是超键，因此能进一步减少冗余与异常。  
但考试中常常先讲 3NF，是因为 3NF 分解通常更便于保持函数依赖，也更容易把“为什么这样分”写清楚。  
因此，若题目强调结构更纯粹，可以继续分析 BCNF；若题目强调依赖保持与答题可解释性，3NF 往往是更稳的第一步。

**本题采分点：**
1. 写出 BCNF 更严格
2. 写出 3NF 更便于依赖保持 / 答题说明
3. 能写出“考试中常先讲 3NF”的原因

**本次已命中的采分点：**
- 提到了“BCNF 分解可能破坏原有依赖关系”这一方向

**本次漏掉的采分点：**
- 没有明确写出 BCNF 为什么“更严格”
- 没有完整写出考试中常先讲 3NF 的原因与场景
- 整体答案只有半句，未形成完整比较

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [x] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- “BCNF 更严格”不是空泛地说更高级，而是要落到：**每个非平凡函数依赖的决定因素都必须是超键**；
- “考试中先讲 3NF”也不是习惯问题，而是因为 3NF 常更便于保持函数依赖、也更容易把答案写完整；
- 这类比较题至少要把“更严格在哪里”“为什么先讲 3NF”两层都写出来。

**标准作答路径：**
1. 先写 BCNF 更严格的判定口径；
2. 再写 3NF 更便于依赖保持与考试表达；
3. 最后写考试中为什么常先从 3NF 讲起。

**纠偏记忆：**
- **比较题至少写两层：定义差异 + 考试表达差异。**

**建议复习动作：**
1. 把“3NF vs BCNF”的区别压缩成 3 句模板并手写 2 遍；  
2. 再用 2 句话回答一次本题，要求必须同时包含“更严格在哪里”和“为什么先讲 3NF”；  
3. 明天做新题前，先口头复述一次：“BCNF 更纯，3NF 更稳、更好写答案”。

### 第29题（下午题）

**题型归属：** 链式无损连接说明

**题目：**  
继续以上分解，请判断该分解是否可以视为合理的无损分解，并写出你的分析思路。

**你的实际作答：**  
- 可以视为合理的无损分解，因为`R1(A, B)` 和 `R2(B, C)` 可以推导出 R1 ，`R2(B, C)` 和 `R3(C, D)` 可以推导出 R2 ，所以可以视为合理的无损分解

**参考答案：**  
该分解可以视为合理的无损分解。分析思路可写为：
1. `R1` 与 `R2` 的公共属性为 `B`，且 `B → C`，因此这一段连接有明确依赖依据；
2. `R2` 与 `R3` 的公共属性为 `C`，且 `C → D`，因此后一段连接同样有依据；
3. 该分解沿 `A → B → C → D` 的依赖链逐段展开，连接时不会无依据地产生伪组合，因此可视为合理无损分解。

**本题采分点：**
1. 给出“合理无损”的结论
2. 写出两段公共属性 `B`、`C`
3. 能结合依赖链说明“逐段连接”思路

**本次已命中的采分点：**
- 给出了“可以视为合理的无损分解”的结论

**本次漏掉的采分点：**
- 没有把两段公共属性 `B`、`C` 单独写出来
- 没有结合 `B → C`、`C → D` 按依赖链逐段说明
- 仍把分析写成“两个子模式能推导出某子表”的不规范口径

**失分原因类型：**
- [ ] 不会知识点
- [x] 会概念但不会下手
- [x] 会分析但写不规范
- [x] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- 链式分解题不能只说“能推出某个子表”；
- 必须按 `R1 ∩ R2 = {B}`、`R2 ∩ R3 = {C}` 这两段公共属性分别说明；
- 再把 `B → C`、`C → D` 与依赖链串起来，才是稳妥写法。

**标准作答路径：**
1. 先写结论：该分解可以视为合理的无损分解；
2. 再写 `R1 ∩ R2 = {B}`，并结合 `B → C` 说明第一段连接；
3. 再写 `R2 ∩ R3 = {C}`，并结合 `C → D` 说明第二段连接；
4. 最后写“该分解沿依赖链逐段展开，因此连接合理”。

**纠偏记忆：**
- **链式无损题，必须写“公共属性 + 依赖链 + 逐段连接”。**

**建议复习动作：**
1. 把第 29 题按参考答案重写一遍，限制在 4 句以内；  
2. 再补 2 道“三表链式分解说明无损”的题，专练公共属性写法；  
3. 与第 25 题并排对照，刻意区分“二元无损判定”和“多表链式说明”的写法差异。

## 三、本次练习暴露出的关键易错点

### 1. 知识判断层面的易错点

- 最高范式判断时，仍可能把“部分依赖”漏判成“已经达到 2NF”；
- 多候选键题还不够稳，容易只找到 `AB`，漏掉 `AC` 这类隐含候选键；
- 对“BCNF 更严格”的表述还停留在感觉层，尚未稳定写成“决定因素必须是超键”这句标准口径。

### 2. 下午题书写层面的易错点

- 链式无损连接说明仍容易写成“两个子表能推出某子表”的模糊表达，而不是“公共属性 + 依赖链 + 逐段连接”；
- 比较题和概括题容易只写半句核心印象，缺少完整的对照结构；
- 候选键题虽然方向更对了，但闭包与最小性还没有形成每题必写的习惯动作。

### 3. 采分点意识层面的易错点

- 这轮已不再是“完全不会”，而是明显进入了**核心结论会了，但关键依据还没写满**的阶段；
- 3NF / BCNF 相关题里，最容易漏掉的是“主属性 / 超键 / 第二候选键”这些决定性依据；
- 模板题较 Day 12 已有改进，但只要无损连接的理由没写规范，采分点还是会掉得很快。

## 四、Day 13 需要继续巩固的重点

1. 最高范式判断的固定顺序：先求候选键，再先看 2NF 能不能过，再谈 3NF / BCNF
2. 多候选键题必须至少验两组可能组合的闭包，防止漏键
3. 3NF / BCNF 判断时，要把“左部是否超键”“右部是否主属性”分开写清
4. 二元无损连接与多表链式无损说明要彻底分开记忆，不混用写法
5. 比较题至少要写出“更严格在哪里 + 为什么考试先讲 3NF”两层
6. 候选键题必须把“闭包 + 最小性”写成固定动作
7. 模板题继续保持“原依赖 → 候选键 → 分解结果 → 保持依赖 → 无损连接 → 结论”的句序稳定性
