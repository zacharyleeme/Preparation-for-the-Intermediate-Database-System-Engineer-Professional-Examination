#### 第19题（下午题）

**题型归属：** SQL 判空查询 / `LEFT JOIN ... IS NULL`  
**你的实际作答：**
```sql
select D.Dno,D.Dname from Dept D left join Emp E on D.Dno=E.Dno where Ename is null;
```

**参考答案：**
```sql
SELECT D.Dno, D.Dname
FROM Dept D
LEFT JOIN Emp E ON D.Dno = E.Dno

```

**本题采分点：**
1. 从 `Dept` 出发；
2. 使用 `LEFT JOIN` 保留全部部门；
3. 用 `E.EmpNo IS NULL` 判断无员工；
4. 输出 `Dno, Dname`。

**本次已命中的采分点：**
- 从 `Dept` 出发；
- 使用了 `LEFT JOIN`；
- 正确输出了 `D.Dno, D.Dname`。

**本次漏掉的采分点：**
- 判空字段应稳定写成 **右表关键字段** `E.EmpNo IS NULL`，你写成了 `Ename is null`；
- 这会把“右表无匹配记录”的标准判断，写成“右表某个普通字段为空”的弱判断。

**失分原因类型：**
- [ ] 不会知识点
- [ ] 会概念但不会下手
- [x] 会分析但写不规范
- [ ] 只写结论没写依据
- [x] 漏步骤 / 漏条件 / 漏关键词
- [ ] 时间不足 / 表达仓促

**正确理解应为：**
- “没有任何员工的部门”这种题，核心不是随便找右表一个字段判空，而是要用**右表关键字段为空**来表达“没有匹配记录”；
- `LEFT JOIN` 方向已经对了，但**判空字段没有锁死成标准壳子**，仍会丢分。

**建议复习动作：**
1. 重写第19题 2 遍；
2. 固定背一句：**判空题先保主表，再判右表关键字段为空**；

## 三、本次练习暴露出的关键易错点

### 1. 判空查询的两套标准壳子还没完全机械化

Day 27 的第19题、第20题说明：
- 你已经知道“没有匹配记录”可以走 `LEFT JOIN ... IS NULL`，也可以走 `NOT EXISTS`；
- 但一到真实落笔：
  - `LEFT JOIN` 版本会把**右表关键字段判空**写松；
  - `NOT EXISTS` 版本会把**相关子查询的关联条件**漏掉。

一句话纠偏：

> **判空题不只是方向对，还要把“右表关键字段”或“内外层关联条件”写完整。**

### 2. “参加过 / 发生过”类统计题，最怕连接方式和计数对象错位

第21题说明：
- 你对 `GROUP BY DeptNo`、`COUNT(DISTINCT Sno)`、`HAVING >= 2` 这些结构已经会了；
- 但“参加社团的学生”这种语义，要求你先保证**统计对象真的发生过关联行为**；
- 否则结构看着像对，结果口径其实变宽了。

一句话纠偏：

> **统计谁，不只看 `COUNT`，还要看连接方式是否真的把对象限定成“发生过这件事的人”。**

### 3. 字段列表对齐问题还在跨题型反复出现

Day 27 的第22题、第23题都说明：
- 目标表字段、视图字段、`SELECT` 字段，仍然需要先对齐再落笔；
- 一旦没有先看清目标结构，就容易：
  - 多写字段；
  - 漏分组维度；
  - 多加不该有的过滤条件。

一句话纠偏：

> **写 `INSERT`、写 `VIEW` 之前，先抄目标结构，再回填查询。**

### 4. “最多统计”题已经会壳子，但内外层口径同步还没完全锁死

第25题说明：
- 你已经知道要用 `HAVING >= ALL (...)` 支持并列最大；
- 当前真正的风险点，变成了：**外层筛了当天，内层却忘了同步筛当天**；
- 这属于“会模板，但模板没写严”。

一句话纠偏：

> **最多统计题的关键，不只是 `ALL`，而是外层和内层的筛选范围完全一致。**
