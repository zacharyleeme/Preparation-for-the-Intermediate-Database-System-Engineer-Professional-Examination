## 三、旧规划问题分析

旧版 `00_整体备考规划.md` 的方向基本正确，但已经不适合作为当前执行版，主要问题如下：

| 问题 | 旧规划表现 | 当前修正 |
|---|---|---|
| 时间基线过期 | 仍以 2026-04-15、剩余 38 天为基准 | 改为 2026-04-30、剩余 23 天 |
| 进度状态过期 | 仍停留在 Day 9 待闭环 | 改为 Day 1～Day 17 已闭环，Day 18 待闭环 |
| 阶段重心偏早 | 仍强调 Day 9 → Day 10 衔接 | 改为真题化、专项化、限时化冲刺 |
| 下午题训练不够密集 | 阶段 B 起每 2 天 1 个下午题小题 | 当前剩余时间必须每天接触下午题采分点 |
| 真题计划不够具体 | 未按年份、题型、产出物拆分 | 改为 2016～2023 精析 + 2024/2025 OCR 补证 + 近年套卷限时 |
| 错题复盘机制仍偏日更 | 有正确率闸门，但未充分结合冲刺 | 改为“日错题 + 周专题错题 + 真题错题 + 采分点错题”四层机制 |
| 练习题来源未充分真题化 | 旧规划强调历年真题，但未固化到每日练习题生成规则 | 新增“后续新练习题优先采用 2016～2023 可靠真题池原题/改编题”正式规则 |

> [!note] 新旧规划衔接结论
> 旧规划中“历年真题 / 高频题型 + 章节补弱 + 小综合训练 + 少量整卷模拟”的总方向继续保留；但从 2026-04-30 起，执行方式必须从“知识点推进”切换为“考试输出能力训练”。

## 九、下午题专项训练计划

### 9.1 五大下午题模板

| 题型     | 必背步骤                                         | 必须产出               |
| ------ | -------------------------------------------- | ------------------ |
| 数据库设计题 | 读需求 → 抽实体 → 抽属性 → 判联系 → 标联系类型 → 转关系模式 → 标主外键 | ER 判断表、关系模式、完整性约束  |
| 范式与分解题 | 列函数依赖 → 求候选键 → 判主属性 → 判范式 → 找冗余/异常 → 分解      | FD 表、候选键、范式判断、分解结果 |
| SQL 题  | 明确表 → 明确条件 → 明确连接 → 明确分组 → 明确子查询/不存在 → 写 SQL | 高频 SQL 写法模板        |
| 恢复题    | 找检查点 → 列事务状态 → 判 Redo / Undo → 推数据值 → 写恢复过程  | Redo/Undo 表、恢复后数据值 |
| 并发题    | 拆事务 → 找冲突 → 判异常 → 判可串行化/一致性 → 提隔离或锁策略        | 调度分析表、改进说明         |

### 9.2 下午题采分点核查表

| 核查项 | 是否完成 |
|---|---|
| 是否写出主键、外键、完整性约束 |
| 是否说明联系类型或联系属性 |
| 是否列出函数依赖，而不是直接跳结论 |
| 是否说明为什么不满足某范式 |
| SQL 是否覆盖题目要求的全部条件 |
| 日志恢复是否明确 Redo / Undo 事务列表 |
| 并发题是否说明冲突或一致性问题 |
| 是否使用考试可得分的短句，而不是泛泛解释 |
