### 5.2 以最近阶段为准的提醒

- **Day 19～Day 20：** 安全设计、备份恢复与运维分析已完成首轮闭环。当前最值得保留的纠偏点是：**最小权限、角色 / 视图边界、RPO / RTO 术语口径、备份 vs 高可用、恢复演练价值、综合收束句**。
- **Day 21～Day 22：** 数据库设计专项已经形成闭环，但 **多值属性拆表、独立记录型关系模式、支付记录唯一标识、订单头 / 明细 / 收货地址边界** 仍应作为数据库设计模板保留。
- **Day 23～Day 24：** 范式专项是当前波动最大的已闭环板块。最需要持续回刷的不是定义本身，而是 **候选键补全、联系属性位置、无损连接与保持函数依赖的分写、桥梁依赖保留、模板句序一致性**。
- **Day 25：** SQL 专项 1 已正式闭环，但当前最需要保留的纠偏点是：**DDL 主键 / 外键 / `CHECK` 顺序、三表连接统一使用显式 `JOIN`、`WHERE NOT EXISTS (...)`、`LEFT JOIN ... WHERE 右表主键 IS NULL`、结果粒度先行、`GROUP BY` 与 `HAVING` 位置稳定**。
